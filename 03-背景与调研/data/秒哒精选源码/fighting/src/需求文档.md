# 需求文档

## 1. 应用概述

### 1.1 应用名称\nK12横版2D格斗小游戏

### 1.2 应用描述
一款面向K12用户的浏览器横版2D格斗小游戏，参考《街头霸王》《拳皇》《大乱斗》但大幅简化。采用Q版卡通风格，提供1v1人机对战，操作简化至3键以内，玩家通过轻拳攻击和格挡防御，先清空对方HP者获胜。

## 2. 用户与使用场景

### 2.1 目标用户
K12阶段学生（6-18岁）

### 2.2 核心使用场景
- 课余休闲娱乐
- 浏览器端快速游玩
- 单人对战AI练习
\n## 3. 页面结构与功能说明\n
### 3.1 页面结构
```\n├── 角色选择页面
├── 对战页面
├── 结算页面
```

### 3.2 角色选择页面
- 展示2个可选角色（不同颜色区分）
- 玩家点击选择其中1个角色
- 显示「开始对战」按钮，点击进入对战页面

### 3.3 对战页面\n\n#### 3.3.1 场景元素
- Q版卡通擂台背景
- 左侧显示玩家角色，右侧显示AI角色
- 顶部显示双方血条（HP）
\n#### 3.3.2 操作控制
- ← 键：角色向左移动
- → 键：角色向右移动
- J 键：轻拳攻击
- K 键：格挡\n\n#### 3.3.3 战斗机制
- 轻拳攻击具备 startup（准备）/active（有效）/recovery（恢复）简化帧感
- 轻拳命中对方时：对方角色闪白+向后击退，扣除HP
- 格挡状态下受到攻击：减伤50%
- AI自动控制对手角色进行移动、攻击、格挡

#### 3.3.4 胜负判定
- 任意一方HP清空时，对战结束
- HP先清空者判负，另一方判胜
\n### 3.4 结算页面
- 显示对战结果（胜利/失败）
- 显示「再来一局」按钮，点击返回角色选择页面
\n## 4. 业务规则与逻辑\n
### 4.1 角色选择规则
- 玩家必须选择1个角色后才能开始对战
- AI角色由系统自动分配（从剩余角色中选择）
\n### 4.2 战斗规则
\n#### 4.2.1 移动规则
- 角色可在擂台范围内左右移动
- 到达擂台边界时无法继续移动
\n#### 4.2.2 攻击规则
- 轻拳攻击范围：角色前方近距离\n- 轻拳命中判定：对方在攻击范围内且未处于格挡状态
- 轻拳伤害：固定数值扣除对方HP
- 攻击动作期间角色无法移动
\n#### 4.2.3 格挡规则
- 格挡状态下角色无法移动和攻击
- 格挡状态下受到攻击时伤害减少50%
- 松开K键解除格挡状态

#### 4.2.4 帧感规则
- startup阶段：角色做出攻击准备动作，此时攻击未生效
- active阶段：攻击判定生效，命中对方则造成伤害
- recovery阶段：攻击结束后角色恢复至可操作状态

#### 4.2.5 击中反馈
- 命中瞬间对方角色闪白（视觉反馈）
- 对方角色向后击退一定距离
\n### 4.3 AI行为规则
- AI自动判断与玩家距离，执行移动、攻击或格挡
- AI行为具备一定随机性，避免过于机械\n
### 4.4 胜负规则
- HP归零时立即判定失败
- 对战结束后自动跳转至结算页面

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 玩家未选择角色直接点击开始 | 「开始对战」按钮置灰不可点击 |
| 双方同时攻击命中 | 双方同时扣血，先归零者判负 |
| 格挡状态下被连续攻击 | 每次攻击均减伤50% |
| 角色移动至擂台边界 | 停止移动，无法越界 |
| 攻击未命中（对方不在范围内） | 攻击动作正常播放，无伤害产生 |
\n## 6. 验收标准

1. 进入游戏，显示角色选择页面，展示2个不同颜色角色\n2. 点击选择其中1个角色，点击「开始对战」按钮
3. 进入对战页面，玩家角色在左侧，AI角色在右侧，顶部显示双方血条
4. 使用← →键移动角色，使用J键发动轻拳攻击AI，命中时AI角色闪白并击退，AI血条减少\n5. 使用K键格挡，AI攻击命中时伤害减半
6. 持续攻击直至AI血条清空，对战结束
7. 显示结算页面，提示「胜利」，点击「再来一局」按钮返回角色选择页面\n
