### 第2讲：设计思维基本知识

#### 2.1 设计思维的历史

| 年份 | 人物 | 贡献 |
|------|------|------|
| 1958 | John E. Arnold | 最早提出工程设计应「以人为本」 |
| 1973 | Robert H. McKim | 《Experiences in Visual Thinking》，强调可视化 |
| 80–90年代 | Rolf A. Faste | 在斯坦福发扬 McKim 方法论 |
| 1987 | Peter G. Rowe | 《Design Thinking》，建筑/城市规划方法论 |
| 1991 | David M. Kelley | 创立 IDEO，设计思维商业化 |
| 2004 | 斯坦福 d.school | 以设计思维为核心教育思想 |

**设计思维由来案例：**
- **Eames 椅**：二战后社会环境，「尽可能把最好的东西带给最多的人」——易组装、低成本、适用于小户型  
- 核心：**满足用户真实需求、关注需求本质、以人为本**

#### 2.2 设计思维的定义（Tim Brown）

> 「设计思维是以人为本的，利用设计师的敏感性以及设计方法，在满足技术可实现性和商业可行性的前提下来满足人的需求的设计精神与方法。」

- IDEO：全球顶尖设计咨询公司（1991年由三家合并）  
- 口号：**像设计师那样思考并实践**

#### 2.3 设计思维六大过程（体系结构）

```
需求理解 → 问题定义 → 思维发散 → 原型设计 → 模型迭代 → 成果发布
         （初步方案）（用户测试）（发布方案）
```

**【必背】各步定义（Acela 火车案例）：**
1. **需求理解**：理解并收集真实需求  
2. **问题定义**：提炼并把握根本问题（本例：购票换票环节过于复杂）  
3. **思维发散**：提出设想并选定最有价值者（更换票务系统）  
4. **原型设计**：进行产品原型设计（重新设计票务系统）  
5. **模型迭代**：投入使用并完善优化（测试新系统）  
6. **成果发布**：正式投入市场（推行新系统）

#### 2.4 经典案例对比

**Airbnb 案例：**
- 传统思维（以数据为导向）→ 在办公室改代码 → 企业濒临崩溃  
- 设计思维（以用户为导向）→ 亲赴纽约拍高质量实景图 → 找到特色短租模式  

**【重点】设计思维要领：**
- Milton Glaser：「关注（attention）这一行为本身让我们真正了解用户需求本质」  
- **以用户为核心**，关注用户痛点  
- 传统思维 VS 设计思维：前者关注表层/数据，后者关注真实需求

#### 2.5 其他思维体系（了解）

- Design Edge：定义、发现、发展  
- Smart Design：设想、创造、完善  
- CMU：定义、发现、综合、建构、精化、反思

### 第3讲：设计思维之需求理解

#### 3.1 什么是需求理解

- 设计思维**第一步**  
- 用户表面行为与潜在需求往往**不一致**  
- 需理解行为与心理、感知联系，把握真实体验  
- 满足最深层需要，弥补设计师对目标用户了解不足  

**案例：插座形式变化** — 更适应用户使用习惯

#### 3.2 移情（Empathy）相关理论

| 学者 | 观点 |
|------|------|
| 费肖尔父子 | 「移情作用」；「移入情感」 |
| 里普斯 | 「审美移情」——自我消融到知觉对象中 |
| Ilpo Koskinen 等 | 《移情设计》——设计前期获得对未来用户的移情理解是关键 |

**移情研究 VS 传统研究：**

| 维度 | 移情研究 | 传统研究 |
|------|----------|----------|
| 用户状态 | 自然生活状态 | 调查访谈、用户参与 |
| 设计师 | 观察、吸引、沉浸 | 专家评估、概念设计 |
| 场所 | 用户场景 | 公司、设计部门 |
| 焦点 | 体验 | 评估、生产、概念 |

**同理心 VS 同情心：** 同理心是「我能明白你的难过」，同情心是「抱歉勾起你的难过」

#### 3.3 需求理解的步骤和方法

| 层面 | 方法 | 要点 |
|------|------|------|
| 第一层面：观察 | 实地观察、类比观察、情景观察 | 之前/现在/之后做了什么，为什么、何时、何地 |
| 第二层面：访谈 | 小组访谈、一对一、专家访谈 | 「三要六不要」；对象：类比/专家/极端用户 |
| 第三层面：沉浸体验 | 模拟场景/身份/心理/行为 | 身临其境、设身处地 |

#### 3.4 经典案例

**拐杖设计：**
- 传统：旁观者观察 → 把手+支撑杆  
- 理解需求后：代入老人身份 → 智能拐杖（照明、摔倒报警、带凳子、防走失）

**盲人智能车窗（福特+Aedo）：**
- 模拟身份：盲人；情景：乘车旅行  
- 本质心理：对视觉的渴望；受阻：视觉缺陷  
- 措施：将窗外景物转为触觉可感知形式（灰度→震动频率）

**【重点】需求理解难点：**
- 对用户深层了解（深度考察、细致划分受众）  
- 对产品精确定位（区分主次需求）  
- 对自身全面认知（自身认知与用户画像融合）  
- 在用户、产品、自身中找到统一解读方式

### 第4讲：设计思维之问题定义

#### 4.1 为什么进行问题定义

- 设计思维**第二步**  
- 在充分理解需求基础上，详细定义正在试图解决的问题  
- **确定用户痛点，明确用户需求，找到真正重要的问题**

**苏联红军探照灯案例（【必背】）：**
- 根本问题：让德军在**看不到**的情况下成功偷袭  
- 而非：如何营造夜黑风高的环境  
- 找到问题根本 → 匹配达成目标的手段 → 事半功倍  

**减脂案例：**
- 「如何变瘦？」→ 过于宽泛  
- 「如何让减脂运动更有效率？」→ 有针对性、有目标性  

#### 4.2 代表人物及观点

| 人物 | 观点 |
|------|------|
| 斯坦福设计学院 | 定义是设计流程不可缺少的一部分 |
| R.G. Gibbs | 问题界定是问题解决者对问题本质的理解 |
| 爱因斯坦 | 「59分钟界定问题，1分钟解决它」 |
| Richard Buchanan | 「设计思维中的问题，定义了以人为本的设计方向」 |
| Jean E. Pretz et al. | 界定问题的挑战在于澄清本质、回答期待解决什么 |
| 杜威 | 「一个界定良好的问题，已经解决一半了」 |

#### 4.3 POV 与 Problem Statement

**POV（Point of View）：**
- 围绕用户需求和愿景重新架构问题  
- 提出具体独特观点  
- 明确目标用户、真实需求、产品愿景  

**Problem Statement（问题阐述）：**
- 包含：观点、设计阈值、用户刚需  
- 三要素：**目标用户、用户需求**（什么时间、什么场景、怎样使用产品）  
- 给设计者足够发挥空间，既不过窄也不过宽  

#### 4.4 问题定义四步骤

1. **Step 1 总结需求**：在需求理解基础上总结  
2. **Step 2 配合工具分析需求**（移情图）：为谁设计？他们需要做什么？  
3. **Step 3 定义观点**：重新定义目标用户、痛点、产品愿景  
4. **Step 4 问题阐述**：按三要素定义问题  

#### 4.5 移情图（Empathy Map）【必背工具】

- 最早由 **XPLANE 公司**开发  
- 六个角度：**听、想、说、关注、痛点、收获**  
- 帮助加深理解、分析用户最关注的问题  

**跑步+音乐案例（完整流程）：**
- Step 1 需求收集：记录运动动作、编辑音乐……  
- Step 2 移情图分析痛点：平台切换、节拍与跑步频率不一致  
- Step 3 观点：运动爱好者；痛点是音乐配合差、运动枯燥；愿景是根据运动动作创作音乐  
- Step 4 问题阐述：如何针对运动爱好者，开发一款能够捕捉运动动作以创作音乐并进行分享的运动社交平台  

#### 4.6 盲道 VS NavCog 案例

- 盲道：没找到问题根本，以设计师需求出发 → 装饰性盲道  
- NavCog：理解用户根本需要 → 3D空间模型+超声波+语音振动导航  
- 根本问题：如何让盲人在行走时知道自己位置并通往正确方向

### 第5讲：设计思维之思维发散

#### 5.1 什么是思维发散

- 设计思维**第三步**  
- 三个空间：**灵感空间**（收集灵感）→ **构思空间**（转变想法）→ **实施空间**（发展计划）  

**开放性思维 VS 封闭性思维：**
- 开放：突破定势，多视角全方位 → 提出更多方案、创造更多选择  
- 封闭：恪守陈规 → 思维定式、思维恒常性  

#### 5.2 代表人物

| 人物 | 观点 |
|------|------|
| 托尼·巴赞 | 发散性思考 = 从中心发散 + 思维爆发；「遇到问题思路越宽越好」 |
| 莱纳斯·卡尔·鲍林 | 「为了有个好主意，必须先有很多想法」 |
| J·P·Guilford | 发散性思维与创造力关系最密切；思维越敏捷灵活，创造力越强 |

#### 5.3 思维发散方法

| 方法 | 要点 |
|------|------|
| **头脑风暴** | 自由联想；臭皮匠协定：不许评价、越多越好、见解无专利、异想天开 |
| **众包** | 用户创造内容；颠覆外包；借社会资源；延伸创新边界 |
| **创意筛选** | 优缺点筛选、根本方向筛选、正反思维、未来性筛选 |

**Threadless 案例：** 粉丝/艺术家设计 → 用户评分 → 头脑风暴+众包+集思广益+创意筛选

**骨传导耳机案例：** 颠覆传统声音传导方式，从使用场景打开脑洞

#### 5.4 思维发散目的

思维发散：创造更多最优解 → 原型设计：快速实践设计方案

### 第6讲：设计思维之原型设计

#### 6.1 什么是原型设计

- 设计思维**第四步**  
- **赋予想法具体外观**，利用模型进行设计构思  
- 通过快速搭建评估、改进想法，推进产品诞生  

#### 6.2 重要性

- 避免重要元素被忽略、阻止不准确假设  
- **降低成本**：原型阶段弥补错误，开发阶段处理成本高  
- **便于沟通**：比文字更直观，易获反馈  

#### 6.3 代表人物

| 人物 | 观点 |
|------|------|
| David Kelley | 「用手来思考」；比抽象思考更有效 |
| Miriam Scheibe | 「想法有多模糊，原型就应该有多简单」 |
| Jake Knapp 等 | Sprint：设计、原型制作、客户验证创意 |

#### 6.4 原型设计三步骤

1. **Step 1 制作模型**：合适媒介、快速搭建，能传递想法即可（不求精致）  
2. **Step 2 讨论模型**：不同场景评估可行性，记录问题  
3. **Step 3 迭代优化**：模型数量减少、精度提高，提炼改进想法  

**IDEO 衣架+白板笔案例：** 外科医生描述「手枪柄」，简陋模型实现跨学科快速沟通

**雅乐轩酒店案例：** 三维虚拟模型 → 用户反馈装潢色调布局 → 节约成本、高效沟通

#### 6.5 原型设计原则

- **快速、粗糙、便宜**  
- **Re-design**：放弃平庸想法，在模型中寻找机会  
- 服务、虚拟体验、组织架构也需原型设计推动想法

### 第7讲：设计思维之模型迭代

#### 7.1 什么是模型迭代

- 设计思维**第五步**  
- 观察真实用户使用，收集反馈，修改调整原型  
- **迭代（Iteration）**：重复反馈过程，每次结果作为下次初始值  

**QWERTY 键盘案例：** ABCD 排列常用字母过近 → QWERTY 拉开常用字母距离 → 需多次试用调整

#### 7.2 模型迭代三过程【必背】

```
测试 → 反馈 → 迭代（循环，测试是关键）
```

**「小怪兽」游戏案例：** 120 个创意遴选 → 50 个原型淘汰 → 5 款用户测试 → 1 款上架

#### 7.3 可用性测试流程

1. 撰写测试脚本（功能罗列、任务设计、场景设置）  
2. 预测试（模拟正式流程）  
3. 正式可用性测试（采访习惯、场景中完成任务、记录过程）  
4. 测试结果分析（按脚本六个维度整理）  

**测试记录：** 5W（Why, When, Where, Who, What）

**IDEO 超市手推车 5 天挑战：** 观察→头脑风暴→制作模型→路人试用→不断迭代

#### 7.4 模型迭代难点

- 明确测试方向限制  
- 观点需来自**用户客观行为**，而非用户自认为的偏爱  
- 团队必须仔细研究结果、设定优先次序、基于原型修改

### 第8讲：设计思维之成果发布

#### 8.1 什么是成果发布

- 设计思维**第六步**

- 将产品投入用户，实现设计价值  
- 检验大众态度，做到**用得上、用得好**

#### 8.3 代表人物

### 第11讲：设计思维访谈（字幕补充）

#### 11.1 嘉宾背景

- 2014 年起接触设计思维，曾从事企业社会责任；后进入青少年教育机构，用设计思维设计 K12 课程与企业培训  
- 强调设计思维是**长期实践**的过程，从单个工具（头脑风暴、用户画像）到完整六步框架  

#### 11.2 如何培养设计思维

- **不是设计师专属**：设计思维是思维方式，需在生活中发现问题、用流程尝试解决（含设计职业生涯）  
- **核心原则**：以人为本、移情——避免「拍脑袋」设计；用原型思维迭代课程/产品  
- **个人变化**：好奇心增强、更爱提问；与固有思维碰撞；需磨练冒险精神与动手能力  
- **方法**：把自己和项目**交给流程**，在实践与迭代中学习  

#### 11.3 国外设计思维教育

- **Stanford d.school**、**HPI（波茨坦）** 为专业设计思维教学院校  
- 美国有 **Design for Educators** 等面向教育者的课程；部分 K12 私校已尝试融入课程体系  
- 中国尚未纳入日常教育体系，但课外机构实践较多  

#### 11.4 三大应用领域

| 领域 | 要点 |
|------|------|
| **服务设计** | 多方共创、系统性思维 |
| **商业创新** | Google、IBM 等用 DT 做产品创新；设有 UX 团队，人本出发 |
| **社会创新** | 用创新方法解决 **Wicked Problem**（复杂、棘手社会问题）；需开放式创新 |

#### 11.5 设计思维优越性（三要素）

1. **移情/以人为本**——从用户角度而非设计者角度  
2. **挑战现有问题**——创新需「退后一步」重新定义问题（盲人摸象：不只看「耳朵」）  
3. **三要素平衡**——用户需求 + 商业可持续性 + 技术可实现性（非天马行空，以产出为目标）  

#### 11.6 经典案例：Embrace 婴儿保温袋（d.school）

- 原需求：为发展中国家设计**低成本保温箱**（传统约 2 万美元）  
- 调研发现：当地妇女**在家生产**，无法带婴儿走很远去医院使用保温箱  
- **aha moment（拍大腿时刻）**：转向**婴儿保温袋**设计——降成本且符合使用场景  
- 启示：深入场景观察、以人为本，才能找到真正创新点  

#### 11.7 发现创意灵感

- 日常**仔细观察**产品/服务使用体验（案例：Snapchat「阅后即焚」抓住「即将消失的信息更珍贵」心理）  
- 建议从**自己喜欢/不喜欢**的产品入手，分析是否体现移情与好设计  

#### 11.8 给同学的建议

- 以**开放心态**持续实践；营造团队信任氛围，增强**创新自信（creative confidence）**  
- 创新 = 打破已知 + 探索未知，设计思维提供可操作的探索路径

### 第12讲：创新设计基础知识

#### 12.1 什么是创新设计**定义：** 发挥创造性思维，将科学、技术、文化、艺术、社会、经济融于设计中，设计具有**新颖性、创造性和实用性**的新产品。

- 三个侧重点：从用户需求出发、以人为本、满足用户需求  
- 一个目标：满足物质、精神需求和生态环保要求  

#### 12.2 设计的进化（设计3.0）

| 时代 | 文明 | 设计形式 | 特点 |
|------|------|----------|------|
| 农耕时代 | 依靠自然资源 | 简单机械工具 | 模仿自然，注重实用 |
| 工业时代 | 机械设备 | 动力系统复杂机械 | 标准化，注重效率 |
| 知识网络时代 | 信息网络 | 数字信息绿色智能系统 | 低碳节能，协同融合 |

**风扇进化：** 车扇风（人力）→ 电风扇（电力）→ 物联网智能风扇

#### 12.3 创新设计内涵五大变化

1. 设计对象：「产品」→「产品、服务和系统」  
2. 问题复杂化：技术更新、新材料、需求激发  
3. 获取用户需求：被动调研 → 分析显性和隐性需求  
4. 设计工程：限制大大降低  
5. 设计模式：众人协同、业余也可参与、群体创新  

#### 12.4 创新设计关键技术

绿色设计、智能设计、全球化设计、定制化/个性化设计、服务设计、品牌设计

#### 12.5 创新设计路径

集成五大构成：**技术、艺术、人本、文化、商业** → 集成

#### 12.6 创新设计特征与趋势【必背】

| 趋势 | 内涵 |
|------|------|
| **绿色低碳** | 节能减排、清洁生产、低碳经济、循环经济 |
| **智能** | 嵌入感知、处理、通信功能部件 |
| **共创分享** | Living Labs、众包协作、跨时空资源整合 |
| **开放融合** | 学科交叉、产业跨界、终端云端融合、跨国人才智力融合 |

### 第13–14讲：人本构成创新设计

#### 13.1 什么是人本构成

- **人本是创新设计的起点**，直接决定设计价值  
- 包含「人－机－环」的需求  
- **Design for All**：以人为本的设计，由人本要素驱动  

**飞利浦儿童核磁共振案例：** 为儿童打造游戏情景、播放音乐，缓解恐惧焦虑

#### 13.2 人本构成三个层次

| 层次 | 关注点 | 时代 |
|------|--------|------|
| **人机工程** | 人和人与物的关系；人的效能和人性价值 | 设计2.0 |
| **人机交互** | 人与计算机信息交换；整个对话过程体验；服务设计 | 设计2.0 |
| **人机融合** | 充分利用人脑和计算机特长；丰富实时数据 | 设计3.0 |

**案例：**
- 人机工程：普通座椅 → 工效学座椅 → 健康座椅  
- 人机交互：天猫无人超市（刷脸、物品识别、无感支付）  
- 人机融合：Peepro 婴幼儿尿意预测（sEMG 技术）  

#### 13.3 马斯洛需求层次（第14讲）

生理需求 → 安全需求 → 社交需求 → 尊重需求 → 自我实现需求  
（需求层次越高，越不容易被满足）

#### 13.4 大数据思维理解需求

| 维度 | 传统调研 | 大数据 |
|------|----------|--------|
| 数据 | 小样本概括整体 | 覆盖所有用户 |
| 分析 | 统计方法 | 对应到人，个性化服务 |
| 功能 | 解决遇到的问题 | 预测未来行为 |

#### 13.5 人本创新设计思维路径

以数据为基础 → 从人机工程/交互/融合角度思考 → 挖掘用户不同层次需求（马斯洛）  
**「人本」是五大构成的源头和出发点**

#### 13.6 与设计思维的联系

- 以人为本、理解用户需求是关键  
- **迭代贯穿整个设计过程**

### 第19–20讲：艺术构成创新设计

#### 19.1 艺术审美 VS 设计审美

- **艺术审美**：蕴含艺术家个人情感和自我意识（书法、绘画）  
- **设计审美**：需面向大众，让大众理解和接受  

#### 19.2 图解思维

- 用速写形式帮助思考的设计表达  
- 草图辅助思考、便于交流，又是下一轮创意起点  
- **流程**：设计原点 → 草图构思 → 交流 → 激发下一轮创意  
- **相关基础**：平面构成、立体构成、色彩构成、设计思维表达  
- 形象思维/直觉思考与逻辑思维**同等重要**（乔布斯：科学无法捕捉排版之美；A&R 听 100 人唱能觉 5 人会成功）

**乔布斯观点：** 形象思考/直觉思考与逻辑思维一样重要；苹果成功因置身于**艺术与科技交汇**之处

#### 19.3 【字幕补充】设计审美三大问题

1. 为什么设计专业出身的设计也会「不够漂亮」？  
2. 为什么设计师自认为漂亮却得不到甲方/用户认可？  
3. 设计审美如何在设计师与普通用户之间**取得一致**？  

**目标人群定位要素**：地域、年龄、文化程度、偏好风格——须先搞清用户审美层面，再表达（反例：用户要本土风，设计师做国际流行范）

#### 19.4 【字幕补充】彼得·希瑞尔案例延伸

- 大众时期：甲壳虫、A4、A6、TT——风格更柔和，符合大众审美  
- 核心理念：**不能只顾及自己的设计感受，更要考虑消费者需求与审美**  
- 2006 年加入濒临破产的**韩国起亚**：面向中国年轻白领，充分考虑**中国市场消费习惯**（进气格栅、腰身、流线）→ 福瑞迪成功  
- **创造审美、引领潮流**：时装周发布款式与配色 → 宣传引领市场；能否被接受还取决于对时尚与消费者心理的判断  

#### 19.5 【字幕补充】培养设计审美

- **多看、多思考、多实践**；从感性认识上升到理性认识  
- 针对特定消费人群与任务，表达美的事物；沟通「产品之美」与「大众消费之美」  
- 审美能力强：对美学敏感、发现生活细微之美、合理表达  

#### 20.1 艺术构成三大作用

1. 吸引眼球  
2. 提升设计品位  
3. 提升产品附加价值  

#### 20.2 艺术构成与品牌

- 宜家：简约、自然、北欧风格  
- 优衣库：百搭、现代、简约、高品质、以人为本  
- 艺术构成 → 品牌竞争力（消费品「低技术」领域提高关键）

#### 20.3 艺术构成在产品生命周期

| 阶段 | 艺术要素作用 |
|------|-------------|
| 导入期 | 尤为注重外观设计，拉大新旧外形差异 |
| 成长期 | 涉及性能、功能、市场细分 |
| 成熟期 | 价值越来越依赖艺术创新设计，成为地位/个性表现 |
| 衰退期 | 注入新吸引力，艺术设计是最好选择 |

### 第21–22讲：商业构成创新设计

#### 21.1 为什么需要商业构成

- 商业模式转变：设计服务 → 设计+销售 → **设计价值最大化**  
- IDEO、Frog、Apple、Amazon、阿里巴巴、Google 等科技公司关注商业构成  

#### 21.2 商业构成两个阶段

| 阶段 | 英文 | 内容 |
|------|------|------|
| **洞察** | Empathize | 始于需求调研 |
| **诠释** | Define | 定义创新方向 |

#### 21.3 市场调研 VS 大数据

**市场调研优点：** 面对面接触，体会内心真实想法，发现隐性需求  
**缺点：** 不能抓住真正购买原因，定性结论有偏差，干扰因素多  

**大数据优点：** 解决传统调研不精准  
**缺点：** 突破性创新的「盲区」；无法发现隐藏背后的东西  

#### 22.1 两种创新策略

| 策略 | 特点 |
|------|------|
| **渐进式创新** | 占创新大多数；体验改变、外观、品质、交互、性能提升 |
| **突破性创新** | 创造全新用户、开发全新市场、构建全新价值网络、设计全新产品 |

**电动自行车案例：** 渐进式（舒适度、重量、续航）→ 突破性（调研开车人对电动车的需求 → 可当电动车又可当平衡车、放后备箱）

#### 22.2 合理运用商业构成三层次

1. 以用户为中心，打造优秀产品和服务  
2. 考虑产品、服务、商业模式、价值网络、竞争壁垒  
3. 综合考虑技术、人本、商业、艺术、文化，跳出以用户为中心框架 → 成功创业

### 第23–25讲：文化构成创新设计

#### 23.1 构成文化的因素
文化、技术、人因、艺术、商业

#### 23.2 三个讨论点

1. **产品设计中的文化表达**（我的产品有文化）  
2. **目标用户和市场的文化理解**（我的用户有文化）  
3. **消费过程的文化建构**（消费过程有文化）  

#### 23.3 文化表达三个层面

| 层面 | 对应设计要素 |
|------|-------------|
| 有形的/物质的 | 材质和视觉 |
| 使用行为的/仪式习俗的 | 功能和交互 |
| 意识形态的/精神层面的 | 体验和品牌 |

**用户体验五层：** 战略层 → 范围层 → 结构层 → 框架层 → 表现层  
（文化表达**不仅**在表现层视觉，而贯穿各层）

#### 23.4 【字幕补充】国家设计文化与品牌文化

- 可研究**德、日、意、美、韩**等国设计文化差异及对中国的启发（政策、产业规划、人才与资源）  
- **品牌文化构建**：在历史变迁、社会变化背景下，从市场竞争出发，用产品/服务/营销打造鲜明品牌文化  
- 产品成功除技术（能用）、艺术（好看）、人本（好用）、商业（能卖）外，**文化**是脱颖而出的关键  

#### 23.5 【字幕补充】「T 恤画龙」反思

- 在 T 恤上画龙 ≠ 有文化这么简单  
- 视觉符号重要，但还需考虑**行为层、精神层、物质层**（材质、款式等）  
- 有形/物质层最重要最有效，但**行为、交互、意识、精神层**往往是文化竞争力的关键  

#### 23.6 【字幕补充】案例：2008 奥运与 From 余杭

**北京奥运会（张艺谋 × 季羡林）：**
- 季老建议「将孔子抬出来」，但开幕式**未**做具象孔子雕像  
- 实际呈现：**有朋自远方来**、**我和你**——含蓄、隐喻式文化表达  

**From 余杭 · 融设计图书馆（品物流形）：**
- 搜集展示中国传统材料；**油纸伞**研究 → 纸椅「漂」  
- 「荣 handmade in 杭州」设计展：竹、丝、土、铜、纸等材质解构再设计  
- **方糖融水**隐喻：文化承载如方糖入水——看不见糖，但尝到甜；图书馆名「融」即此意  
- 文化表达须**从用户与市场出发**，非设计师自我欣赏

#### 24.3 【字幕补充】次文化 / 亚文化

  1. **小社群不小**——小众用户联网形成亚文化群体，汇聚成庞大市场（**长尾理论**：冷门需求总和可超主流）  
  2. **新文化更多更快**——快手/抖音推动新群体（如海底捞网红调料、吃鸡、奇葩说等 IP）

## 附录 F：名词与方法辨析（期末易混考点）

> **用途**：针对客观题中「概念区分、工具归属、步骤顺序」类陷阱；表述以课件 JSON 与 RAG 练习题库为准。

### F.1 设计思维六步 vs 各步工具

| 设计思维步骤 | 核心目标 | 典型工具（归属本步） | 易混点 |
|--------------|----------|----------------------|--------|
| **需求理解** | 观察用户、建立移情 | 观察、访谈、沉浸体验、问卷调查 | 移情是**思想**，观察/访谈/沉浸是**方法** |
| **问题定义** | 找到真正重要的问题 | POV、HMW、用户旅程图、鱼骨图、5why | **5W 不属于思维发散**（单元测试四） |
| **思维发散** | 产生大量创意 | 头脑风暴 → 思维导图 → 价值链分析（**递进**关系） | 价值链用于**筛选**，不是发散起点 |
| **原型设计** | 低成本快速验证方案 | 纸模、Axure/Mockplus/XD、3D 打印 | 强调**短时间、低成本**，非精致完善 |
| **模型迭代** | 测试—反馈—改进循环 | 可用性测试、AB Test | 迭代循环：**测试-反馈-迭代** |
| **成果发布** | 传播与打动用户 | 故事叙述法、现场演示法 | 发布步骤：**交代背景-故事讲述-升华主题** |

### F.2 需求理解：移情 vs 同情 vs 观察层次

| 概念 | 含义 | 考点 |
|------|------|------|
| **移情（Empathy）** | 站在用户角度理解真实需求；需求理解核心思想 | 源于美学；**不是**里普斯首次用于设计思维（单元测试一） |
| **同情（Sympathy）** | 情感共鸣但未必理解需求 | 课件强调设计思维要移情而非仅同情 |
| **需求理解三层次** | 观察 → 访谈 → 沉浸体验 | 由浅入深（单元测试一） |

### F.3 问题定义：POV vs 问题描述 vs HMW

| 术语 | 作用 | 结构/要点 |
|------|------|-----------|
| **POV（观点陈述）** | 从用户视角凝练洞察 | [用户] + [需求] + [洞察] |
| **问题描述** | POV 的进一步具化 | **观点、设计阈值、用户刚需**（单元测试二） |
| **HMW（How Might We）** | 将问题转为可发散的挑战 | 以「我们如何……」引导创意 |
| **问题定义四步** | 总结需求 → 工具分析 → 定义观点 → 问题阐述 | 单元测试二标准答案 |

### F.4 思维发散三工具辨析

| 工具 | 阶段角色 | 逻辑关系 |
|------|----------|----------|
| **头脑风暴** | 前期获取大量创意 | 臭皮匠协定：不许评价、越多越好、见解无专利、异想天开 |
| **思维导图** | 梳理、组织创意 | 围绕核心辐射连接 |
| **价值链分析法** | 筛选创意 | 从企业与产品价值角度评估；**不属于**纯发散工具 |

**聚思维模型顺序**：发散 → 筛选 → 聚合 → 细化（单元测试三）

### F.5 原型 vs 迭代 vs 发布

| 概念 | 关键词 | 易错 |
|------|--------|------|
| **原型** | 低成本、便于沟通理解 | 不要求与成品一致材料；**非**精致完善 |
| **原型小循环动力** | 用户反馈 + 设计优化 | 非「问题定义+思维发散」 |
| **可用性测试** | 设计任务 → 招募 → 执行 → 观察记录 | 顺序题高频 |
| **AB Test** | 两版本线上对比 | 属模型迭代工具 |
| **成果发布** | 产品生产 + 宣传推广 | 非仅「投入市场」 |

### F.6 设计 1.0 / 2.0 / 3.0

| 阶段 | 时代 | 特点 | 关联 |
|------|------|------|------|
| **设计 1.0** | 农耕/传统 | 模仿自然、实用 | 简单机械 |
| **设计 2.0** | 工业/现代 | 标准化、效率 | 人机工程、人机交互 |
| **设计 3.0** | 知识网络/创新 | 低碳智能、协同融合 | 人机融合、绿色智能 |

### F.7 创新设计五大构成辨析

| 构成 | 核心定位 | 易混 |
|------|----------|------|
| **人本构成** | 创新设计**起点**；Design for All | **直接决定设计价值**（易与「技术决定价值」混淆） |
| **技术构成** | 产品化、商品化关键；价值体系**基础性** | 是人本创新的**基础前提**（方向勿反） |
| **艺术构成** | 审美、品位、附加价值 | 吸引眼球但**不**以增加实用性为主 |
| **商业构成** | 商业模式、市场 | 商业模式创新地图：**创造、转换、整合、平衡** |
| **文化构成** | 文化内涵、传承创新 | 与人本/艺术有交叉但侧重文化符号与意义 |

**五大构成标准集合**：技术、艺术、文化、人本、商业（顺序可变，**无**社会/数字/产业构成）
