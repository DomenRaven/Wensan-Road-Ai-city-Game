# TPL-07 · 生活模拟 · 设计规范与实现路径

> **slug**：`life_sim` · **参考**：4399 做饭/种菜风 · **D10**：L0

---

## 一、品类概述

K12 生活模拟锁定 **3 步任务链**：获取 → 加工 → 交付。参考 4399《美食烹饪家》《Township》简化循环，**点击热区**操作，无复杂经济。

---

## 二、核心设计规则

### 2.1 资源循环（Source → Converter → Sink）

```
种植/点击获取 ──► 加工（计时条）──► 交付顾客/完成订单 ──► 金币/星星
         ▲___________________________________|
                    解锁新物品（L0 固定）
```

### 2.2 做饭三部曲（4399 通用）

| 阶段 | 操作 | 反馈 |
|------|------|------|
| Prep 备菜 | 点击食材 → 切/洗 | 进度条 |
| Cook 烹饪 | 拖拽到灶台 | 绿=完美，红=焦（L0 可无失败） |
| Serve 上菜 | 拖成品到顾客 | 金币 + 表情 |

### 2.3 种菜简化链

```
点击空地 → 选种子 → 等待 N 秒 → 点击收获 → 送入厨房
```

### 2.4 模拟经营核心（机核 Township 拆解）

- **乐趣在分配瞬间**，不在重复点击
- 前期慢、后期快 → K12 单局 **2 分钟** 完成 3 单即可
- 升级 = 缩短烹饪时间（主要 tuning 旋钮）

## 三、参数规格

### core 层

| 键 | 值 |
|----|-----|
| `task.chain_steps` | `3` |
| `interaction.mode` | `click_hotspot` |
| `economy.mode` | `order_based` |
| `fail.mode` | `forgiving` |

### tuning 层

| 键 | 默认 | 范围 |
|----|------|------|
| `crop.grow_time_sec` | 8 | 5–15 |
| `cook.time_sec` | 5 | 3–10 |
| `order.count_to_win` | 3 | 2–5 |
| `order.patience_sec` | 30 | 20–45 |
| `reward.coins_per_order` | 10 | 5–20 |

## 四、Godot 实现路径

```
templates/life_sim/
├── core/
│   ├── hotspot_manager.gd      # Area2D 点击
│   ├── task_chain.gd           # 状态：get/process/deliver
│   ├── timer_bar.gd            # ProgressBar 烹饪
│   └── order_queue.gd
├── scenes/room.tscn            # 单房间等距/侧视
```

1. 每个可交互物 `Area2D` + `input_event`
2. `task_chain` 有限状态机，UI 箭头提示当前步
3. 顾客：`Timer` 耐心值，超时仅少给分（不 scary）
4. 无存档；每局 reset

## 五、K12 约束

- 食物：汉堡、沙拉、卡通蛋糕
- 无火焰烧伤特效；焦了 = 灰色贴图
- 操作 ≤2 键（点击 + 可选取消）
