# TPL-11 · 雷霆战机（纵版 STG）· 设计规范与实现路径

> **slug**：`shmup` · **视角**：纵版 · **D10**：L0 + L1

## 一、品类概述

纵版射击（雷电/1942 风）：玩家船在屏下，敌从上方出现，**弹幕密度可控**。K12 降低密度，强调可读性。

---

## 二、核心设计规则

### 2.1 弹幕设计（Danmaku）

| 概念 | 说明 |
|------|------|
| 空间密度 | 单位面积子弹数 |
| 时间密度 | 节奏与 pacing |
| 负空间 | 必须留可 dodge 通道 |
| Toaplan Pattern | 敌从屏幕**对侧**交替出现，迫使移动 |

**原则**：子弹多 ≠ 难；结构 + timing 更致命（Sparen Guide A4）。

### 2.2 弹幕类型（L0 子集）

| 类型 | 实现 |
|------|------|
| 直线齐射 | 固定角度 n 发 |
| 瞄准单发 | 角度 = atan2(player - spawn) |
| 扩散扇 | 中心角 ± spread |
| 慢速流弹 | 高密度但可闭眼划过去（教学用） |

### 2.3 难度 metric（学术参考）

```
Coverage = 被任意子弹覆盖的屏幕格占比（8×8 网格）
```

K12 L0：`Coverage < 0.35`，避免满屏。

### 2.4 核心循环

```
波次敌机 → 击破得分/道具 → Boss（可选）→ 下一章
```

## 四、Godot 实现路径

```
templates/shmup/
├── core/
│   ├── player_ship.gd
│   ├── bullet_pool.gd
│   ├── pattern_spawner.gd      # 弹幕 Resource
│   ├── enemy_wave.gd
│   └── scroll_background.gd
├── data/patterns/              # JSON 参数序列
```

1. 背景 `ParallaxBackground` 或 UV scroll
2. 子弹池：`Array[Node2D]` 复用
3. `pattern_spawner`：每帧 `F(t|p) → bullet params`（Keiki 论文模型）
4. 玩家 hitbox **小于** 精灵（1–2px 核心）
