# TPL-04 · 塔防 · 设计规范与实现路径

> **slug**：`tower_defense` · **视角**：顶视网格 · **D10**：L0 + L1

---

## 一、品类概述

经典路径塔防：敌人沿**预设路径**行进，玩家在路径旁放塔防守。K12 简化：单路径、2–3 种塔、5 波敌人。

---

## 二、核心设计规则

### 2.1 网格与放置

| 规则 | 说明 |
|------|------|
| 网格 | 16×12 或 12×10，`cell_size=64px` |
| 放置 | `path_adjacent_only` 路径邻格可建 |
| 路径验证 | 放塔后 BFS/A* 验证 spawn→exit 仍有路，否则拒绝 |
| 卖塔返还 | 50% 金币 |

### 2.2 数值公式（core 锁定）

```
damage = max(1, tower_atk - enemy_def)
range  = circle_tile 格数
```

### 2.3 波次 scaling

```
HP(w) = HP_base × (1.18)^(w-1) + (w × 10)
```

或分离函数：`health(w)` + `count(w)` 独立曲线（GameDev SE）。

### 2.4 经济循环

```
击杀得金 → 建塔/升级 → 清波奖励 → 下一波
```

## 三、参数规格

### core 层

| 键 | 值 |
|----|-----|
| `grid.cell_size` | `64` |
| `grid.placement_rule` | `path_adjacent_only` |
| `path.mode` | `preset_waypoints` |
| `combat.damage_formula` | `max(1, atk - def)` |
| `wave.scheduler` | `interval_spawn` |

### tuning 层

| 键 | 默认 | 范围 |
|----|------|------|
| `economy.start_gold` | 120 | 80–200 |
| `tower.basic.damage` | 15 | 5–40 |
| `tower.basic.range` | 3 | 2–5 |
| `tower.basic.cost` | 50 | 30–80 |
| `enemy.basic.hp` | 40 | 20–100 |
| `enemy.basic.speed` | 60 | 30–120 |
| `wave.count` | 5 | 3–8 |
| `wave.spawn_interval_ms` | 800 | 400–1500 |

## 四、Godot 实现路径

```
templates/tower_defense/
├── core/
│   ├── grid_map.gd
│   ├── path_follower.gd        # Path2D + progress
│   ├── tower_placer.gd
│   ├── path_validator.gd       # BFS 放塔后验路
│   └── wave_scheduler.gd
├── data/waypoints.json
```

1. `TileMap` 标记 path/grass/buildable
2. 塔：`Area2D` 每帧检测范围内敌人列表，Timer 开火
3. 波次：`[{type, count, interval}]` 队列 spawn
4. UI：点击 grass 邻路径格 → 扣金 → 实例化塔
