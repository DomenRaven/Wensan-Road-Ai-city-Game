# TPL-02 · 横版格斗 · 设计规范与实现路径

> **slug**：`fighting` · **视角**：侧视 2D · **D10**：L0 preset

---

## 一、品类概述

K12 横版格斗采用**极简 2 键对战**：轻攻击 + 防御/闪避，无连招表复杂度。参考 Street Fighter 帧数据思想，但**大幅简化**为固定 3 招 + 机器人 AI。

---

## 二、核心设计规则

### 2.1 帧数据三阶段（60 FPS）

| 阶段 | 典型帧数 | 说明 |
|------|----------|------|
| Startup | 4–8 | 出招前摇，可被打断 |
| Active | 2–4 | hitbox 生效窗口 |
| Recovery | 8–16 | 收招硬直 |

### 2.2 Hitbox / Hurtbox

- 每动画帧存储 hitbox/hurtbox 矩形（相对角色原点）
- K12 L0：**每招仅 1 个 hitbox**，全帧同一区域（降低制作量）
- 碰撞：`active` 阶段 hitbox 与对手 hurtbox 重叠 → 扣 HP + hitstun

### 2.3 核心循环

```
回合开始 → 移动接近 → 攻击/防御 → 命中扣血 → HP 归零 → 下一回合
```

### 2.4 K12 简化规则

| 全功能格斗 | K12 L0 |
|------------|--------|
| 6+ 键位 | **≤3 键**：左移、右移、攻击（防御长按攻击键） |
| 复杂连招 | 最多 2 段 cancel |
| 血腥特效 | Q 版星星、机器人对战 |
| 10+ 角色 | 2 角色 preset |

## 三、参数规格

### core 层

| 键 | 值 |
|----|-----|
| `combat.frame_rate` | `60` |
| `combat.hitbox_mode` | `per_move_static` |
| `combat.rounds` | `best_of_1` |
| `ai.mode` | `reaction_timer` |

### tuning 层

| 键 | 默认 | 范围 |
|----|------|------|
| `player.max_hp` | 100 | 50–150 |
| `move.light.startup_frames` | 6 | 4–10 |
| `move.light.active_frames` | 3 | 2–5 |
| `move.light.damage` | 12 | 8–20 |
| `move.light.hitstun_frames` | 12 | 8–20 |
| `player.walk_speed` | 180 | 120–260 |

## 四、Godot 实现路径

```
templates/fighting/
├── core/
│   ├── fighter.gd              # 状态机：idle/walk/attack/hit/stun
│   ├── frame_data.gd           # Resource: startup/active/recovery
│   ├── hitbox_manager.gd       # Area2D 开关
│   └── round_manager.gd
```

1. `AnimationPlayer` 驱动动画；`_physics_process` 递减帧计数器
2. `state==ATTACK && frame in active_range` → 启用 Hitbox Area2D
3. AI：距离 < 阈值 50% 攻击，否则逼近；受击后 0.5s 不进攻
4. 相机：固定侧视，不 zoom
