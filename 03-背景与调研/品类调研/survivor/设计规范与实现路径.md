# TPL-03 · 割草肉鸽 · 设计规范与实现路径

> **slug**：`survivor` · **视角**：顶视 · **D10**：L0 + L1

---

## 一、品类概述

参考 Vampire Survivors 类 **horde survivor**：玩家移动躲避，武器**自动攻击**，杀怪掉经验 → 升级三选一 → 撑到时间结束。单局 **3 分钟**，K12 无恐怖元素。

---

## 二、核心设计规则

### 2.1 核心循环（螺旋式）

```
移动躲避 → 自动杀怪 → 捡经验宝石 → 升级选能力 → 敌人密度上升 → 终局 Boss 波
```

后期循环**语义变化**：前期求生存 → 中期求 build → 后期割草（Lost Attic Games）。

### 2.2 时间与密度

| 参数 | K12 L0 建议 |
|------|-------------|
| 单局时长 | **180s** |
| 初始敌密度 | 低，3–5 只/屏 |
| 密度曲线 | 线性或 `game_progress_ratio` 曲线 |
| 升级间隔 | 前 30s 每 15s 一级，之后加快 |

### 2.3 武器/auto-attack

- 最近敌人方向发射 / 环形弹幕
- 冷却由 `weapon.interval_ms` 控制，**禁止**玩家手动瞄准（≤2 键：移动）

### 2.4 升级系统（L0 固定表）

- 3 选 1：+伤害 / +移速 / +拾取范围 / 新武器槽
- L1 可改 `theme` 武器名与图标

## 三、参数规格

### core 层

| 键 | 值 |
|----|-----|
| `combat.auto_target` | `nearest_enemy` |
| `xp.pickup_magnet` | `true` |
| `level.up_choice_count` | `3` |
| `spawn.pattern` | `ring_around_player` |

### tuning 层

| 键 | 默认 | 范围 |
|----|------|------|
| `session.duration_sec` | 180 | 120–240 |
| `player.speed` | 200 | 140–280 |
| `weapon.base_damage` | 10 | 5–20 |
| `weapon.interval_ms` | 800 | 400–1500 |
| `enemy.base_hp` | 15 | 8–40 |
| `spawn.rate_per_sec` | 1.5 | 0.5–3.0 |
| `xp.gem_value` | 5 | 3–15 |

## 四、Godot 实现路径

```
templates/survivor/
├── core/
│   ├── player_survivor.gd
│   ├── auto_weapon.gd          # Timer + 方向计算
│   ├── xp_gem.gd
│   ├── level_up_ui.gd          # 暂停选升级
│   └── horde_spawner.gd        # 环形生成
```

1. 敌人生成：玩家周围随机角度 `radius 400–600` 屏外
2. 经验：`Area2D` 磁吸 `lerp` 向玩家
3. 对象池：敌人/子弹/宝石复用（VS 数百实体）
4. 难度：`spawn.rate = base + progress * curve.sample(ratio)`

## 五、K12 约束

- 敌人：史莱姆、糖果、小机器人
- 无死亡血腥；HP 归零显示「时间到」或重试
- 单局 3min 硬上限
