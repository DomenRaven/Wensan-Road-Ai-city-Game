# TPL-05 · 跑酷 · 设计规范与实现路径

> **slug**：`parkour` · **视角**：侧视/轨道 · **D10**：L0

---

## 一、品类概述

无尽跑酷：**自动向前**移动，玩家仅跳跃/换道躲避障碍。参考 Subway Surfers / Temple Run 的**受控随机**障碍 prefab 库。

---

## 二、核心设计规则

### 2.1 障碍生成（Controlled Randomness）

| Prefab 类型 | 说明 |
|-------------|------|
| Single | 单障碍，最常见 |
| Short | 2–4 连续障碍 |
| Long | 长段挑战 |
| Branch | 分岔（K12 可省略） |
| Trick | 奖励/陷阱 |

**禁止**纯随机单块堆砌 → 用 prefab 库 + 权重表。

### 2.2 难度曲线

```
progress = 1.0 - (time_left / game_length)   # 0→1
spawn_interval = curve.sample(progress)       # 如 2.0s → 0.25s
obstacle_weight_hard = lerp(0.1, 0.6, progress)
```

### 2.3 车道（可选）

- 单跑道 L0：仅跳跃
- 三跑道：`x ∈ [-2, 0, 2]`，左右切换 + 跳跃

### 2.4 核心循环

```
自动跑 → 障碍逼近 → 跳跃/换道 → 距离得分 → 碰撞结束
```

## 三、参数规格

### core 层

| 键 | 值 |
|----|-----|
| `movement.mode` | `auto_run_forward` |
| `obstacle.spawn_rule` | `distance_based` |
| `score.mode` | `distance_time` |

### tuning 层

| 键 | 默认 | 范围 |
|----|------|------|
| `runner.speed` | 240 | 160–360 |
| `jump.velocity` | -420 | -300 ~ -520 |
| `obstacle.density` | 0.6 | 0.3–1.0 |
| `obstacle.min_gap_px` | 180 | 120–280 |
| `round.duration_sec` | 60 | 30–90 |

## 四、Godot 实现路径

```
templates/parkour/
├── core/
│   ├── auto_runner.gd
│   ├── obstacle_spawner.gd     # Timer + prefab 权重
│   ├── scroll_world.gd         # 背景/地面反向移动
│   └── prefabs/                # obstacle_set_*.tscn
```

1. 玩家 `CharacterBody2D` 固定 X 或三车道 lerp
2. 障碍从 `spawn_z` 生成，`-speed * delta` 移动
3. `VisibleOnScreenNotifier` 离屏 `queue_free`
4. 保留**可通行带宽**：生成前 agent 检测或 prefab 预验证
