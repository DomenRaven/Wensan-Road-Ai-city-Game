> **产品版本**：`1.1`（tag `v1.1`）

> **关联方案**：[`模板引擎/6.26_甲方升级方案与LLM评估_v1.0.md`](./模板引擎/6.26_甲方升级方案与LLM评估_v1.0.md)

## 一、架构概览

```text
浏览器 (Chrome/Edge 全屏)
  ├─ Kiosk 静态页  :8080  →  /kiosk/  ·  /kiosk/edu/
  └─ FastAPI 后端  :8000  →  会话 · 生成 · play/launch · actions 轮询
         │
         └─ 外置 Godot 4.6 窗口（Win32 自动贴右半/下半屏）
```

| 组件 | 是否必须 | 说明 |
|------|----------|------|
| Python 3.11+ | ✅ | 后端 + 静态 HTTP 服务 |
| Godot 4.6 Standard | ✅ | B6/B7 试玩 launch |
| Redis 5.x | 推荐 | 会话持久；不可用时可内存降级 |
| Nginx | 可选 | 局域网单端口反代（`kiosk_lan_nginx.ps1`） |
| Node.js | ❌ | Kiosk 为纯静态 JS，运行时不需要 |
| Playwright | ❌ | 仅开发/E2E 验收用 |

## 二、硬件与系统

| 项 | 建议 |
|----|------|
| 操作系统 | **Windows 10/11 64 位**（Godot 窗口归位依赖 Win32） |
| 显示器 | 横屏 1920×1080 展厅大屏；竖屏 pad 1080×1920 可选 |
| 内存 | ≥ 8 GB（Godot + 浏览器 + API 同机） |
| 磁盘 | ≥ 2 GB 可用（含 `workspace/` 会话工作区） |
| 网络 | 本机环回即可；多终端访问需局域网 + 反代 |

**非 Windows**：Kiosk 与 API 可运行，但 **Godot 外置窗自动贴边不可用**（`window_placed=false`，需手动拖窗）。

## 六、环境变量

复制模板并编辑：

```powershell
copy backend\.env.example backend\.env
```

| 变量 | 必须 | 说明 | 示例 |
|------|------|------|------|
| `GODOT_PATH` | ✅ | Godot **GUI** 可执行文件路径（非 console 版优先） | `F:\Godot\...\Godot_v4.6.3-stable_win64.exe` |
| `REDIS_URL` | 推荐 | Redis 连接串 | `redis://127.0.0.1:6379/0` |
| `ALLOW_MEMORY_FALLBACK` | 可选 | Redis 不可用时用内存会话 | `true` |
| `DEPLOYMENT_SERVER_OS` | 可选 | 部署标注 | `Windows` |
| `DEPLOYMENT_TERMINAL_LAYOUT` | 可选 | 横/竖屏布局标注 | `landscape` |

**注意**

- 修改 `config/creative_templates` 或 `code_anchors` 后须 **重启 API**。
- `GODOT_PATH` 可指向目录；launcher 会自动解析 `Godot_v4.6.3-stable_win64.exe`。
- **勿将** 含真实 API Key 的 `.env` 提交 Git。

## 七、Redis（推荐）

```powershell
# 一次性：下载便携 Redis 到 tools/redis/
.\05-工具脚本\install_redis.ps1

# 每次开机（可与 backend 同启）
.\05-工具脚本\run_redis.ps1
```

验证：`GET http://127.0.0.1:8000/health` → `session_backend` 为 `redis`。

未装 Redis 且 `ALLOW_MEMORY_FALLBACK=true` 时，会话存内存，**重启 API 会丢失会话**。

## 八、展厅启动

### 8.1 一键启动（推荐）

**首次打包**（需 Python 3.11+ 与 PyInstaller，仅做一次）：

```powershell
cd E:\文三路AI馆\2.ai生成游戏
.\05-工具脚本\build_launcher.ps1
```

生成：

| 文件 | 说明 |
|------|------|
| `启动游戏工坊.exe`（仓库根目录） | 双击启动，推荐展厅使用 |
| `05-工具脚本\dist\启动游戏工坊.exe` | 同上，构建输出目录 |

**双击 `启动游戏工坊.exe` 将自动：**

1. 启动 Redis（若已 `install_redis.ps1` 且未运行）
2. 创建/复用 `backend/.venv` 并启动 API `:8000`
3. 启动 Kiosk 静态服务 `:8080`
4. 打开浏览器 → `http://127.0.0.1:8080/kiosk/edu/`（B 链教育版）

关闭黑色控制台窗口或按 **Enter** 即停止全部后台服务。

```powershell
# A 链快玩入口
.\启动游戏工坊.exe --mode fast

# 仅启动服务、不自动开浏览器
.\启动游戏工坊.exe --no-browser
```

**前提**：本机已安装 Python（首次会自动建 venv）；`backend/.env` 中 **GODOT_PATH** 须指向 Godot 4.6 GUI 可执行文件（试玩必需）。

## 九、局域网部署（可选）

同一 Wi‑Fi 下 pad/副屏访问本机 Kiosk：

```powershell
# 需本机已安装 nginx
.\05-工具脚本\kiosk_lan_nginx.ps1 -Port 9080 -BackendPort 8000
```

访问：`http://<本机局域网 IP>:9080/kiosk/edu/`

若 API 与 Kiosk 不同源，须同步修改 [`config/kiosk_edu_spec.json`](../config/kiosk_edu_spec.json) 中 `api_base` 为实际 API 地址。

| 路径 | 说明 |

## 十三、常见故障

| 现象 | 排查 |
|------|------|
| launch 503 · GODOT_PATH | 检查 `backend\.env` 路径 · 模板 `project.godot` 存在 |
| Godot 启动即退出 | 用 GUI 版 exe · 命令行 `Godot --path templates/platformer` 试开 |
| `window_placed=false` | 非 Windows · HWND 未找到 · 仍可从状态卡手动找窗 |
| 配方/锚点改了不生效 | 重启 `run_backend.ps1` |
| 浏览器跨域 / API 连不上 | `kiosk_edu_spec.json` 的 `api_base` 与 API 地址一致 |
| Redis 连接失败 | 运行 `run_redis.ps1` 或保持 `ALLOW_MEMORY_FALLBACK=true` |
| WinError 32 写 workspace | 关闭占用中的 Godot · 重启 API |
