## 三、部署架构（展厅 · 单机联网）

```text
                    ┌─────────────────────────────────┐
  10× 展厅终端       │  展陈服务器（待定 OS/云/物理机）   │
  浏览器/Kiosk ─LAN─►│  FastAPI + Redis (Session×10)   │
  或轻客户端         │  Godot 游戏实例池 / 导出包分发     │
                    │  MinIO（产物）· nginx 反代         │
                    └─────────────────────────────────┘
```

| 组件 | 职责 |
|------|------|
| **FastAPI** | Session 创建/销毁、品类路由、排队、健康检查 |
| **Redis** | 活跃 Session ≤10、队列、锁、心跳 |
| **Godot Runtime** | 每 Session 独立 `workspace/{id}`；优先 **Windows exe 子进程** 或 **Godot headless + 流式/本地窗** |
| **nginx** | HTTPS/HTTP 反代；静态 Kiosk 前端 |
| **MinIO** | 导出包、二维码、上墙素材 |

**「单机游戏、联网访问」释义**：游戏本体为 **Godot 单机 exe/进程**（无联机对战）；观众通过 **展厅局域网** 访问服务器选游戏、拉 preset、启动本地/远程渲染实例。

**服务器规格（10 并发建议）**：

| 项 | 最低 | 推荐 |
|----|------|------|
| CPU | i7-13700 | i9-13900 / Ryzen 9 |
| GPU | RTX 4060 8GB | **RTX 4070 12GB+** |
| RAM | 32GB | **64GB** |
| 磁盘 | 1TB NVMe | 2TB NVMe |
| 网络 | 千兆 LAN | 千兆 + 独立 VLAN |

> 10 路 Godot 同时运行压力极大；**D10 策略**：同时 **试玩 ≤6**、其余 **排队/观战**；或 **2 台服务器** 各承载 5 Session（备选）。

| # | 项 | 标准 |

## 六、风险与裁剪（10 天必读）

| 风险 | 对策 |
|------|------|
| 11 类全量 AI 创作做不完 | D10 仅 **L0 preset**；L1 限 5 类 |
| 10 路 Godot GPU 不够 | 试玩并发 cap=6 + 队列；或双机 |

| 生活模拟 scope 大 | 单房间 **3 步任务链**（种→收→做） |
| 用户要「融合玩法」 | **S2 玩法子模式** + 跨品类跳转（见 `play_variants.json`） |
