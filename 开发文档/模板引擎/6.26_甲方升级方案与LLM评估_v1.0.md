### 2.2 横竖屏自适应（R2）· ✅ 已交付

| 甲方要求 | v1.0 基线 | v1.1 实现 | 主要文件 |
|----------|-----------|-----------|----------|
| 横屏布局 | 固定 45/55 双栏 | `landscape` 左右双栏 grid | `orientation.js` · `dual-pane.js` · `edu-styles.css` |
| 竖屏布局 | 简单 media 叠行 | `portrait` 上下双行 · B1–B2 单栏 | 同上 |
| 检测 | 无 | `body[data-orientation]` · breakpoint 900 · 150ms 防抖 | `orientation.js` |
| 顶栏 | 固定三列 | 竖屏折行 · 作品名省略 | `edu-styles.css` |

**断点规则（与实现对齐）**

```css
/* landscape：宽 ≥ 900 且 orientation landscape，或宽 ≥ 1200 */
body[data-orientation="landscape"] .dual-pane-root { grid-template-columns: minmax(45%,1fr) minmax(0,55%); }

/* portrait：orientation portrait 或 宽 < 900 */
body[data-orientation="portrait"] .dual-pane-root {
  grid-template-columns: 1fr;
  grid-template-rows: minmax(42%,1fr) minmax(0,58%);
}
```

### 2.3 Godot 试玩面板分区（R3）· ✅ 已交付

| 甲方要求 | v1.0 基线 | v1.1 实现 | 主要文件 |
|----------|-----------|-----------|----------|
| 横屏占右半屏 | 默认位置 | `godot_zone_rect` 屏幕绝对坐标 → Win32 贴右栏虚线区 | `play.py` · `godot_window_layout.py` · `edu-wizard.js` |
| 竖屏占下半屏 | 默认位置 | Win32 `get_monitor_bottom_half_rect` · 主显示器下半 | 同上 |
| launch 载荷 | 空 body | `getViewportPayload()` · `window_placed` 降级文案 | `orientation.js` · `play/launch` |

**Launch 请求扩展（与 backend 对齐）**

```json
{
  "orientation": "landscape",
  "client_viewport": {
    "screen_x": 0,
    "screen_y": 0,
    "screen_w": 1920,
    "screen_h": 1080,
    "kiosk_rect": { "x": 960, "y": 0, "w": 960, "h": 1080 }
  }
}
```

后端 `GodotLauncher.launch()` 在 `subprocess.Popen` 后 `sleep(0.5)`，调用 `place_godot_window(pid, kiosk_rect)`。

**降级**：非 Windows · 归位失败 · 无 PID → 仅保证 kiosk 内布局正确，状态卡文案提示「请在大屏右侧/下方找到游戏窗口」。

**说明**：展陈 P0 已选型 **外置 Godot**（非浏览器 embed）。R3 的「面板」= **外置游戏窗口 + kiosk 内对应半区引导**，二者同步分区。

## 四、大模型（LLM）接入评估

### 4.1 结论

| 问题 | 答案 |
|------|------|
| **本次 UI/响应式/Godot 布局是否必须接 LLM？** | **否** |
| **整个 B 链是否已完全依赖 LLM？** | **否** — v1.0 已展陈可演示 |
| **建议在方案中预留 LLM 吗？** | **是** — 作为 P2 增强，不挡 P3 UI 上线 |

### 4.2 功能触点对照（与 6.24 规格一致）

| 阶段 | 功能 | v1.0 实现 | 是否需 LLM | 若接入 LLM 的位置 | 优先级 |
|------|------|-----------|------------|-------------------|--------|
| B1 | 意图「我想打飞机」→ 品类 | `POST /intent/match-genre` · `intent_genre_lexicon.json` 关键词 | **否**（可选增强） | 低置信时调用 LLM 分类或生成追问句 | P2-01 |
| B2 | 推荐游戏名 | `GET /creative/name-suggestions` 预制列表 | **否** | 可按意图生成创意名（非必须） | P2-05 |
| B4 | 完型填空 | `creative_templates` + `tuning_path` 映射 | **否** | 禁止 LLM 猜 tuning 路径 | — |
| B5 | 需求分析 | `analyze-requirements` **preset-only** | **否**（复杂定制时需要） | `resolution=llm_patch` 写 workspace 白名单 | P2-02 |
| B5 | 代码剧场滚动 | `assets/code_theater/{slug}.jsonl` 预制 | **否** | 展示非实时 LLM 流（避免长等待） | 不做 |
| B5 | generate/v2 | 本地 merge `game_config` | **否** | llm_patch 成功后再 merge | P2-02 |
| B6–B7 | launch / 高亮 | Godot + `.edu_actions.jsonl` | **否** | — | — |
| **P3** | 蓝白/横竖屏/Godot 位 | 纯前端 + Win32 | **否** | — | **本期** |

### 4.3 推荐给甲方的 LLM 路线（可选）

```text
首期（P3）：不接 LLM，按期交付蓝白 + 响应式 + Godot 分区。
增强（P2）：① B1 低置信 LLM 兜底  ② B5 复杂句子 llm_patch（仍不改 core）
展后：语音 B1 · 流式剧场（若甲方单独立项）
```

**环境与合规**：LLM 需甲方提供 API Key / 内网网关；`backend/.env` 不入库；超时 **8s** 降级 preset（已有 6.24 技术规范 §八）。

## 五、风险与回退

| 风险 | 缓解 |
|------|------|
| Godot 窗口归位偶发失败 | 降级文案 + 讲解员手动拖窗；不阻塞试玩 |
| 竖屏代码区过高 | portrait 上栏可折叠「只看游戏」切换钮（P3 可选） |
| 蓝白对比度不足 | 代码高亮改用深蓝底 `#1e3a5f` 块（代码区可保留浅灰底+深字） |
| 改动影响冻结品类 | 仅改 `kiosk/**` · `config/kiosk_edu_spec.json` · `backend` launch 层 |

**回退**：`git checkout v1.0` 或仅回退 `kiosk/edu` 与 spec 色板。
