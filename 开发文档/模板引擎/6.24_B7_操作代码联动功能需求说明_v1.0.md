# 6.24 · B7 操作代码联动功能需求说明 v1.0

> **阶段**：B7 · 试玩 · 操作 ↔ 代码高亮（**教育核心**）  
> **配置**：`config/code_anchors/{slug}.json`  
> **API**：`POST /sessions/{id}/play/action` · `GET /sessions/{id}/play/actions`  
> **前端**：`kiosk/edu/code-highlight.js`

---

## 1. 功能概述

儿童在右侧 Godot 试玩时，每次关键操作（跳、射击、得分等）触发左侧代码区 **自动滚动 + 高亮对应行**，并可选口语气泡解释。

**教学目标**：建立「我按下的操作 ↔ 哪段代码/参数」的直观联系。

---

## 2. 用户故事

> 我跳起来时，左边代码里「jump_velocity」那一行会亮起来，并告诉我「你选得越高，跳得越猛！」

---

## 3. action_id 枚举（首期）

### platformer（P0 ≥2，目标 3）

| action_id | 触发时机 | 锚点 | 气泡文案示例 |
|-----------|----------|------|--------------|
| `jump` | 玩家起跳 | `tuning.player.jump_velocity` | 跳跃力度：你选得越高，跳得越猛！ |
| `stomp_enemy` | 踩死敌人 | `core/player_platformer.gd` 踩怪逻辑 | 从上方踩到敌人就能消灭它！ |
| `collect_coin` | 拾取金币 | `tuning.scoring.coin` | 每枚金币加分就在这里设定！ |

### 其他品类（P1 各 ≥3）

| slug | action_id 示例 |
|------|----------------|
| shmup | `fire` · `hit` · `pickup` |
| survivor | `move` · `level_up` · `pickup_xp` |
| pingpong | `rally` · `score` · `power_smash` |
| fighting | `light_punch` · `block` · `special` |
| parkour | `jump` · `slide` · `collect_coin` |
| racing | `steer` · `boost` · `lap_complete` |

## 4. code_anchors 结构

```json
{
  "anchors": {
    "jump": {
      "file": "config/game_config.json",
      "path": "tuning.player.jump_velocity",
      "line_hint": 12,
      "caption": "跳跃力度：你选得越高，跳得越猛！"
    },
    "stomp_enemy": {
      "file": "core/player_platformer.gd",
      "path": null,
      "line_hint": 120,
      "caption": "从上方踩到敌人就能消灭它！"
    }
  }
}
```

`generate/v2` 完成后，`code_map` 写入真实 `line` 号。

---

## 5. Godot 上报桥

### P0 方案

1. generate 时复制 `templates/_edu/edu_action_bridge.gd` 到 workspace
2. 各品类在 **workspace 副本** 事件处调用 `EduActionBridge.emit_action("jump")`
3. 桥写 append-only：`workspace/{id}/.edu_actions.jsonl`

```json
{"action_id":"jump","t_ms":1719200001}
```

### 上报 API

- Godot 写文件后，前端轮询 `GET /sessions/{id}/play/actions?since={cursor}`
- 或 Godot HTTP `POST .../play/action`（P1）

**约束**：桥失败不崩溃游戏 · 不修改 templates 原件

## 6. 前端高亮行为

| 步骤 | 行为 |
|------|------|
| 1 | 收到 `action_id` |
| 2 | 查 `code_map[action_id]` |
| 3 | 切换左栏到对应 `file` |
| 4 | `scrollIntoView` 到 `line` |
| 5 | 添加 `.code-line-highlight` 2s |
| 6 | 显示气泡 `caption` 3s |

### 样式

```css
.code-line-highlight {
  background: #fef08a;
  border-left: 4px solid #f59e0b;
  animation: pulse-highlight 1.5s ease-in-out 1;
}
```

- 等宽 ≥14px · 行高 1.6
- 禁止频闪（animation 单次）

### 去抖

同一 `action_id` 500ms 内只高亮一次。

## 7. 验收标准

| # | 标准 |
|---|------|
| 1 | platformer 跳跃必触发高亮 |
| 2 | platformer 踩怪触发高亮 |
| 3 | 高亮行与 `game_config` 实际值一致 |
| 4 | 滚动动画流畅 · 不遮挡右栏游戏 |
| 5 | P1 每品类 ≥3 action 有效 |
| 6 | 讲解员可让儿童复述对应关系 |

---

## 8. 非功能需求

- 轮询间隔 200ms（可配置）
- action 日志仅 session 生命周期内保留
- release 后删除 `.edu_actions.jsonl`
