## 二、架构速查（排障地图）

```text
Godot workspace
  templates/_edu/{genre}_hooks.gd     → 监听游戏信号
  templates/_edu/edu_action_bridge.gd → 写 .edu_actions.jsonl
        ↓
FastAPI  GET /sessions/{id}/play/actions?since=
        ↓
kiosk/edu/code-highlight.js           → 轮询 · 优先级 · 展示锁
        ↓
kiosk/edu/code-viewer.js              → 切文件 · 滚行 · 黄框 captionBubble
```

| 层级 | 职责 | 常见故障点 |
|------|------|------------|
| **hooks** | 把玩法 moment 映射为 `action_id` | 信号未接上 · 异步载入场景 · **高频动作刷屏** |
| **bridge** | 去抖 + 追加 jsonl | 同 action 500ms 内合并 |
| **anchors** | `config/code_anchors/{genre}.json` | `action_id` 重复覆盖 · 行号/snippet 错 |
| **code_map** | `generate/v2` 产出 | 旧 session 无新锚点 |
| **前端** | 优先级 / suppress / 拉文件 | JS 语法错误整页挂 · move/fire 占线 |

**硬规则**：hooks 与 bridge 在 **B5 `generate/v2`** 时复制进 `workspace/{session_id}/`；**只刷新网页不会更新 hooks**。

## 三、共性根因：「占线动作」

### 3.1 现象

- 击毁敌机 / 吃道具 / 吸经验时，左侧**不滚到对应 `.gd`**，或黄色解说框**刚弹出就被顶掉**  
- 控制台无 API 报错，Godot 游戏本身正常

### 3.2 根因模式

某 `action_id` 在 `_physics_process` 或子弹池轮询中 **每帧/每发** 上报，B7 轮询间隔仅 200ms，`handleAction` 反复：

1. `fetchWorkspaceFile` 切回「占线」文件  
2. `highlightLine` + `showCaption` 重置展示锁  

教育上重要的低频动作（`kill_enemy`、`pickup_xp`）永远排不上队。

### 3.3 标准修复三板斧

| 手段 | 落点 | 适用 |
|------|------|------|
| **停报或首报** | `templates/_edu/{genre}_hooks.gd` | 自动射击 `fire`、持续移动 `move` |
| **优先级 + 展示锁** | `kiosk_edu_spec.json` → `code_highlight.action_priority` · `code-highlight.js` | 吸经验 10 > 杀怪 8 > move 1 |
| **suppress_actions** | 同上 | 仅 `fire` 等彻底不需要试玩高亮的动作 |

推荐优先级参考：

```json
"action_priority": {
  "fire": 1, "move": 1,
  "kill_enemy": 8,
  "pickup": 10, "pickup_xp": 10, "level_up": 10,
  "hit": 7
}
```

## 四、分品类问题 · 修复实录

### 4.1 乒乓 pingpong

| # | 问题 | 根因 | 修复 |
|---|------|------|------|
| P1 | 行号 gutter 被裁切 | `.code-scroll` 垂直居中 + gutter `overflow` | `edu-styles.css`：`align-items:flex-start` · gutter `overflow:visible` |
| P2 | 有解说词但代码不高亮 | 锚点在 `core/ball.gd`，viewer 只读了 `game_config` | `GET .../workspace/file?rel_path=` · `code-highlight.js` 按 `action_id` 先拉文件再高亮 |
| P3 | JSON 有技能开关但玩法未生效 | 技能需 `core/` 接线（展陈范围外） | **未改 core**；文档标注为 backlog |

**经验**：B7 多文件高亮的前置条件是 **workspace 文件 API + 前端按锚点 `file` 字段切文件**。

---

### 4.2 飞机 shmup

| # | 问题 | 根因 | 修复 |
|---|------|------|------|
| S1 | 自动开火一直占线 | 子弹池 `active` 边沿每发上报 `fire` | hooks **移除 fire 上报**；`suppress_actions: ["fire"]` |
| S2 | 击毁敌机不高亮 `enemy_ship.gd` | 仅连 spawner；场景异步载入 | hooks v2：`enemy.destroyed` 直连 + `EnemySpawner` 备援 + **0.4s 重扫** |
| S3 | 展厅只有 1 命太难 | `max_hp: 1` | `game_config.json` + `player_ship.gd` 默认 **3 命**（用户显式要求） |
| S4 | `code_map` 偶发缺锚点 | 旧 session / 生成前 anchors | `getMergedCodeMap()` 保底 · B7 演示钮「打敌机/吃道具/受伤」 |
| S5 | **实机吃道具无高亮/黄框**（2026-06-26） | ① `kill_enemy` 展示锁压 `pickup`(9) ② hooks deferred 接线竞态 ③ tuning `pickup` 盖 `apply_powerup` | `pickup` 优先级 **10** · `_wire_single_powerup` 即时接线 · `getMergedCodeMap` 保留 core · `play/launch` 同步 hooks |

**锚点**：`kill_enemy` → `core/enemy_ship.gd` · `destroyed.emit(...)` · `pickup` → `core/player_ship.gd` · `apply_powerup`  
**生效**：重新 B5 **或** B7 点「开始试玩」（launch 会 `apply_edu_workspace_patch` 覆盖 hooks）。

---

### 4.3 割草 survivor

| # | 问题 | 根因 | 修复 |
|---|------|------|------|
| V1 | 吸经验无高亮/解说 | `move` 每帧上报（注释写「首次」实现却是持续） | hooks：`move` **仅首次**；`suppress_actions` 含 `move` |
| V2 | 锚点不直观 | 指向 `survivor_arena._on_gem_collected` | 改为 **`xp_gem.gd:72`** `collected.emit` |
| V3 | 宝石池未接线 | Game 晚于 hooks `_ready` 载入 | `node_added` + `xp_gem` 组 + 树遍历 + **3 次重扫** |
| V4 | 受击无高亮 | hooks **从未上报 `hit`** | `hp_changed` 降血 → `hit` · `take_hit` 锚点 `player_survivor.gd:125` |
| V5 | 前端锚点被旧 code_map 覆盖 | `getMergedCodeMap` 后者覆盖前者 | **fallback 优先 file/caption** · workspace 失败走 preview API |

**锚点**：`pickup_xp` · `level_up` · `kill_enemy` 见 `config/code_anchors/survivor.json`  
**生效**：重新 B5。

---

### 4.4 全局 · B0 卡死（非单品类）

| 问题 | 根因 | 修复 |
|------|------|------|
| 欢迎页一直「正在准备展厅…」 | `code-highlight.js` `setCodeMap` 内多余 `};` → **整文件 SyntaxError** | 重写 `setCodeMap` 块；`node --check kiosk/edu/code-highlight.js` |
| 连锁错误 | `EduCodeHighlight` 未挂载 → `init()` 调 `configure` 抛错 | 修语法后自愈 |

**经验**：任一 `kiosk/edu/*.js` 语法错误会导致 **整链 B0 阻断**；发版前必跑 `node --check`。

## 五、新品类 B7 接入检查清单

### 5.1 后端 / 模板（不改 core）

- [ ] `config/code_anchors/{genre}.json`：每个讲解 moment 有 `action_id` · `file` · `snippet_gd` 或 `path`  
- [ ] `templates/_edu/{genre}_hooks.gd`：信号与 `action_id` 一一对应  
- [ ] `GENRE_HOOKS` 含该 slug · `apply_edu_workspace_patch` 测过  
- [ ] **无高频 action**；移动/连射仅首报或不上报  
- [ ] 游戏场景异步载入时有 **deferred 重扫**（建议 0.4s）

### 5.2 前端

- [ ] `getGenreHighlightFallback(slug)` 保底 `code_map`  
- [ ] `action_priority` 中教育重点动作 ≥ 9  
- [ ] B7 品类演示钮（讲解员备用）  
- [ ] `node --check` 通过

### 5.3 验收（每款 3 分钟）

1. 重新开始 → 走完 **B5**  
2. B6 `play/launch` → B7  
3. Godot 内触发目标 moment → 左栏滚到正确文件行 + 黄框 ≥3s  
4. 点演示钮 → 同上（验证前端链路）  
5. 底部 log 无 `code_map 缺少 action_id` · 无 `无 workspace`

## 六、排障决策树

```text
高亮完全不工作？
├─ B0 卡死 → node --check code-highlight.js / edu-wizard.js
├─ 无 workspace → 重跑 B5 generate/v2
├─ Godot 未写 jsonl → workspace 内是否有 EduHooks + EduActionBridge autoload
└─ 有 jsonl 无 UI → code_map 是否含 action_id · 前端是否轮询（B7 + workspacePath）

高亮一闪就没 / 错文件？
├─ 是否有 fire/move 占线 → hooks 减报 + priority
├─ 锚点 file 是否与当前 viewer 一致 → workspace/file API
└─ 行号是否 snippet 对得上 → build_code_map_for_workspace 实测

展厅难但功能对？
└─ tuning（如 shmup max_hp）改 config；core 默认值仅作兜底
```
