## A.SKILLS-shmup-CORE · 战机技能 Godot 接线

**模型**：Composer  
**前置**：用户口头解冻 `templates/shmup/core/**`  
**FRS**：[`6.26_SKILLS_shmup_小技能需求说明_v1.0.md`](./6.26_SKILLS_shmup_小技能需求说明_v1.0.md)

```text
【6.26 SKILLS · 窗 SKILLS-shmup-CORE · 战机小技能 Godot 接线 · v1.0】

仓库：E:\文三路AI馆\2.ai生成游戏
会话范围：仅 templates/shmup/core/**（用户已解冻 shmup core）

## 目标
B4 勾选的 bomb / laser_beam 在 Godot 试玩可感知；未勾选时按 skill 键无效果。
技能 ID 与 UI 严格一致：bomb · laser_beam（optional_skills + creative_templates）。

## 必读
- 开发文档/模板引擎/6.26_SKILLS_shmup_小技能需求说明_v1.0.md
- 开发文档/模板引擎/6.26_SKILLS_技术与执行规范_v1.0.md
- templates/shooter/core/player_shooter.gd（接线范本）
- templates/shmup/core/skills/bomb.gd · laser_beam.gd
- templates/shmup/core/bullet_pool.gd（clear_enemy_bullets · pierce）
- templates/shmup/core/player_ship.gd

## 交付
1. player_ship.gd
   - preload BombSkill / LaserBeamSkill（或内联路径）
   - _physics_process：两技能 tick(delta)
   - _handle_skills()：Input.is_action_just_pressed("skill")
   - bomb 成功：bullet_pool.clear_enemy_bullets() + 音效
   - laser active：_fire_bullet 使用 spawn_player_bullet(..., pierce=true)
   - 双技能同键：document 优先级（建议先 bomb 后 laser）

2. skills/bomb.gd（若缺 execute 副作用，可增 static func on_activated() 由主脚本调 pool）

3. 本地验证 game_config：
   - enabled_skills: []
   - enabled_skills: ["bomb"]
   - enabled_skills: ["laser_beam"]
   - enabled_skills: ["bomb","laser_beam"]

## 验证
godot-mcp run_project 或本地打开 templates/shmup 无 ERROR
git diff 仅限 templates/shmup/
python 05-工具脚本/freeze_genre_snapshot.py write && verify

## 硬禁止
- 改 kiosk/** · code_anchors（属 UI 窗）
- 改 optional_skills.json / creative_templates skill_ids
- 改 templates/pingpong 或其它品类
- 动 templates/shmup/core 移动/命中公式（非技能门控）

## 收工
- 新建：评审记录/6.26_SKILLS_shmup_CORE_验收记录.md（待用户签字）
- 勾选：6.26_SKILLS_整合任务清单 SK-SH-C01~07
- 更新：工作状态记录 §八
- 告知用户：CORE 人工验通过后开 SKILLS-shmup-UI
```
