## 一、技术定位（30 秒版）

**6.24 = B 链教育重构**：在已验收的 **7 款 A 深 R2 Godot 模板** 之上，增加 **NLU 选类 · 完型填空 · 双栏代码剧场 · 操作↔代码联动**，不改变 `templates/` 原件。

```text
用户自然语言 + 填空答案
    → analyze（preset 优先 · llm_patch 备选）
    → copytree templates/{slug}/ → workspace/{session_id}/
    → merge game_config.json（±30% clamp）
    → 左：代码展示/高亮    右：Godot 试玩
```

---

## 二、架构分层

| 层 | 职责 | 6.24 新增/变更 |
|----|------|----------------|
| **Kiosk Edu UI** | B0–B7 步骤机 · 双栏 · 剧场 · 高亮 | `kiosk/edu/` 新路由 |
| **FastAPI** | session · creative · intent · generate/v2 | `routers/intent` · `creative` · `generate` v2 |
| **Creative 服务** | 填空解析 · analyze · code_map 构建 | `backend/app/services/creative/` |
| **Workspace** | copytree · config merge · 路径守卫 | 延续 `workspace_guard` |
| **Godot 模板** | 运行时 · action 信号 | workspace 内可选 debug 桥（不改 templates 原件） |
| **配置资产** | 填空/锚点/剧场/词表 | `config/creative_templates/` 等 |

**禁止跨层违规**

- UI 不得直接写 `workspace/` 文件系统
- LLM 不得绕过 `analyze-requirements` 直写磁盘
- 单次 Cursor 会话：仅改 **一个品类** 的 `creative_templates` + `code_anchors` + `code_theater`，或仅改 `kiosk/edu`，或仅改 `backend` 一层

## 四、API 契约（实现必遵）

### 4.1 保留 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/bootstrap` | 启动校验 · orphan 清理 |
| POST | `/sessions` | 创建会话 |
| POST | `/sessions/{id}/play/launch` | 启动 Godot |
| POST | `/sessions/{id}/release` | 释放 workspace |

### 4.2 新增 API

#### `POST /intent/match-genre`

```json
// Request
{ "text": "我想玩马里奥那种", "session_id": "uuid" }

// Response
{
  "matched_genre": "platformer",
  "confidence": 0.92,
  "reply_text": "听起来你想玩横版闯关！",
  "candidates": []
}
```

- P0：关键词 + `intent_genre_lexicon.json`
- P2：低置信（<0.6）返回 `candidates` 最多 3 个

#### `GET /creative/name-suggestions?genre={slug}`

```json
{ "genre": "platformer", "suggestions": ["星星大冒险", "跳跳小子", "管道奇遇", "金币猎人"] }
```

#### `GET /creative/templates/{genre}`

返回 JSON Schema 风格表单（见 B4 功能说明）。

#### `POST /sessions/{id}/creative/answers`

```json
{ "answers": { "q_move": "fast", "q_jump": "high", "q_enemy": "default" } }
```

校验：`question_id` 合法 · `option_id` 属于该题 · 写入 `session.payload.creative_answers`

#### `POST /sessions/{id}/analyze-requirements`

```json
// Response
{
  "resolutions": [
    { "question_id": "q_jump", "resolution": "preset", "tuning_path": "tuning.player.jump_velocity", "value": -440 }
  ],
  "llm_patch_required": false,
  "code_map_preview": { "jump": { "path": "tuning.player.jump_velocity", "caption": "..." } }
}
```

#### `POST /sessions/{id}/generate/v2`

```json
// Response
{
  "ok": true,
  "workspace_path": "workspace/{uuid}/",
  "config_path": "workspace/{uuid}/config/game_config.json",
  "code_map": { },
  "genre": "platformer"
}
```

与 v1 `/generate` 差异：

| 项 | v1 `/generate` | v2 |
|----|----------------|-----|
| 前置 | 需 `recap_confirmed` | 需 `creative_answers` + analyze 完成 |
| 输入 | wizard S2–S7 payload | creative_template 映射 |
| 输出 | workspace 路径 | + `code_map` |

v1 **保留**供旧向导；教育版 **仅调 v2**。

#### `POST /sessions/{id}/play/action`

```json
{ "action_id": "jump", "timestamp_ms": 1719200000000 }
```

Godot 调试桥或轮询代理上报；前端据此滚动高亮。

## 六、Creative Template 配置规范

文件：`config/creative_templates/{slug}.json`

```json
{
  "version": "1.0",
  "genre": "platformer",
  "display_name": "横版闯关",
  "name_suggestions": ["星星大冒险", "跳跳小子", "管道奇遇", "金币猎人"],
  "questions": [
    {
      "id": "q_move",
      "widget": "single_choice",
      "prompt_template": "你希望主角移动得【{options}】",
      "options": [
        { "id": "slow", "label": "慢一点", "tuning_path": "tuning.player.move_speed", "value": 170, "code_anchor_id": "move" },
        { "id": "default", "label": "刚刚好", "tuning_path": "tuning.player.move_speed", "value": 200, "code_anchor_id": "move" },
        { "id": "fast", "label": "快一点", "tuning_path": "tuning.player.move_speed", "value": 240, "code_anchor_id": "move" }
      ]
    }
  ]
}
```

**规则**

1. `tuning_path` 必须存在于该 slug 的 `game_config.json` 或 `tuning_feel_overrides.json` 可推导路径。
2. `value` 应用时经 `clamp_tuning_value`（±30% 相对模板默认值）。
3. `code_anchor_id` 必须在 `config/code_anchors/{slug}.json` 有定义。
4. 技能题引用 `config/optional_skills.json` 的 `id`，写入 `tuning.enabled_skills`。

## 七、代码锚点与剧场

### 7.1 `config/code_anchors/{slug}.json`

```json
{
  "anchors": {
    "jump": {
      "file": "config/game_config.json",
      "path": "tuning.player.jump_velocity",
      "line_hint": 12,
      "caption": "跳跃力度：你选得越高，跳得越猛！",
      "snippet_gd": null
    },
    "stomp_enemy": {
      "file": "core/platformer_player.gd",
      "path": null,
      "line_hint": 84,
      "caption": "踩到敌人头顶就会消灭它！",
      "snippet_gd": "func _stomp_enemy ..."
    }
  }
}
```

- `line_hint`：生成 workspace 后由后端扫描文件刷新真实 `line` 写入 `code_map`
- `snippet_gd` 可选：用于左侧展示 core 片段（只读）

### 7.2 `assets/code_theater/{slug}.jsonl`

每行一条：

```json
{"delay_ms": 0, "text": "extends CharacterBody2D", "highlight": false}
{"delay_ms": 80, "text": "var jump_velocity = -400", "highlight": true, "anchor_id": "jump"}
```

从 `templates/{slug}/core/*.gd` 与 `game_config.json` **摘录**，与 anchors 同源。

## 八、analyze 决策树（实现）

```text
FOR each answer in creative_answers:
  IF option has tuning_path → resolution=preset, queue merge
  ELIF option is skill_id → resolution=preset, append enabled_skills
  ELIF free_text AND complexity>threshold → resolution=llm_patch (P2)
  ELSE → resolution=preset, use default option

IF any llm_patch:
  CALL llm with JSON schema contract
  ON timeout/fail → strip llm_patch, use preset defaults, flag degraded=true

BUILD code_map FROM code_anchors + merged config
RETURN analyze_result
```

P0：**禁止** `llm_patch` 分支；`llm_patch_required` 恒为 `false`。

## 九、前端双栏规范

| 区域 | 技术 | 约束 |
|------|------|------|
| 左栏 | 虚拟文件树 + `<pre>` 代码区 | 只读 · 等宽 ≥14px · 行号 |
| 右栏 | iframe / 子窗口 / 外链 Godot | contain 缩放 · 参考 racing 540×960 |
| 顶栏 | 三段：创作 / 制作 / 试玩 | 触控按钮 ≥48px |
| 高亮 | CSS class `.code-line-highlight` | `#fef08a` 底 + 左侧竖条 + pulse 2s |

状态机：`idle` → `theater` → `applying` → `ready` → `playing`

## 十、Godot Action 桥（B7）

**P0 方案（推荐）**：workspace 副本内增加 `core/edu_action_bridge.gd`（通过 generate 时从 `templates/_edu/` 复制），游戏事件 emit 后写：

`workspace/{session_id}/.edu_actions.jsonl`

后端或前端轮询 `GET /sessions/{id}/play/actions?since=` 获取。

**约束**

- 桥脚本 **不写入** `templates/{slug}/` 原件；仅 workspace 副本
- action_id 枚举与 `code_anchors` 一致
- 上报失败不阻塞游戏

## 十一、隔离与鲁棒性（延续 6.23）

| 检查点 | 实现位置 |
|--------|----------|

| 阶段 | workspace 是否存在 |
|------|-------------------|
| B0–B4 | 否 |
| B5 generate 起 | 是 |
| release 后 | 否 |
