## 三、实现要点（修复窗执行）

### 3.1 抽取共享函数（`edu-wizard.js`）

```text
renderGenreDemoActionsHtml(genre, { compact, launched })
bindGenreDemoActions(genre, { launched })
```

- **B6** `renderPlayReadyPanel`：演示区调用上述函数，**删除**写死 jump/stomp。
- **B7** `renderB7Play`：同上；`launched === true` 且 `genre === 'shmup'` 时不渲染「受伤」。

### 3.2 各品类演示 action_id（与 anchors 一致）

| genre | 按钮文案 | action_id |
|-------|----------|-----------|
| platformer | 跳！ / 踩怪 / 捡金币 | jump · stomp_enemy · collect_coin |
| pingpong | 击球 / 得分 | rally · score |
| shmup | 打敌机 / 吃道具 | kill_enemy · pickup ·（B7 启动后无受伤） |
| survivor | 消灭 / 吸经验 / 升级 | kill_enemy · pickup_xp · level_up |
| fighting | 轻拳 / 重拳 / 格挡 / 大招 | light_punch · heavy_punch · block · special |
| parkour | 跳跃 / 滑铲 / 捡金币 / 吃道具 | jump · slide · collect_coin · pickup_powerup |
| racing | 撞车 / 撞路障 / 完圈 | hit_npc · hit_trap · lap_complete |

### 3.3 CSS（`edu-ambient.css` + 必要时 `edu-styles.css`）

- 右栏内容外包 `.pane-right-stack`：`width:100%` · `max-width:min(100%,480px)` · `margin:0 auto`
- `body.edu-ambient-dual .pane-right-inner:has(.pane-right-stack)` → `justify-content:flex-start` · `overflow-y:auto` · `padding-top:16px`
- `.demo-panel-actions--wrap`：`gap:10px` · 钮 `flex:1 1 calc(50% - 5px)` · `min-width:120px` · 防挤出右栏

### 3.4 冻结遵守

- **不改** `templates/{slug}/core/**` · `code_anchors/*.json` · `_edu/*_hooks.gd`（pingpong/shmup 冻结）
- 仅前端演示钮 → `simulateAction(action_id)`，与既有 B7 桥一致

## 四、验收标准（每品类抽测）

路径：`/kiosk/edu/` → B0–B5 → **B6 未点启动** → 目视演示区：

1. 按钮**品类正确**（非 platformer 默认钮）
2. 按钮在右栏**卡片内对齐**，无横向溢出、无压到左栏
3. 点钮 → 左侧代码**黄框高亮**对应行
4. 点「开始试玩」→ B7 → 演示钮仍正确；shmup **无「受伤」**
5. `node --check kiosk/edu/edu-wizard.js`
6. 修 pingpong/shmup 后：`python 05-工具脚本/freeze_genre_snapshot.py verify`（若触及冻结共享文件则先用户解冻）

**建议抽测 slug**：pingpong · shmup · survivor · fighting（覆盖用户 2.1–2.7）
