# 6.24 · B1 意图匹配功能需求说明 v1.0

> **阶段**：B1 · 「今天你想玩什么游戏呢？」  
> **关联 API**：`POST /intent/match-genre`  
> **配置**：`config/intent_genre_lexicon.json`  
> **映射**：[`6.24_文件映射表_v1.0.md`](./6.24_文件映射表_v1.0.md) §二

---

## 1. 功能概述

将儿童一句自然语言映射到 **7 款已验收 slug** 之一，并返回口语化确认文案，写入 session。

---

## 2. 用户故事

> 作为展厅小朋友，我说「想打飞机」，系统应听懂并带我去制作 **街机飞机射击** 游戏。

---

## 3. 界面需求

| 元素 | 要求 |
|------|------|
| 主文案 | 「今天你想玩什么游戏呢？」 |
| 输入框 | 单行 · max 80 字 · placeholder 轮播示例句 |
| 示例芯片 | 可点填入：「马里奥闯关」「打飞机」「割草打怪」 |
| 提交 | 大按钮「开始匹配」· ≥48px |
| 加载 | 匹配中显示轻快 loading（≤2s） |
| 低置信 | 展示最多 3 张候选品类卡供点选（P1+） |

**禁止**：用下拉框代替自然语言（与旧 S1 区分仅在低置信兜底时出现卡片）。

---

## 4. 匹配域

| slug | 显示名 | 意图关键词（示例） |
|------|--------|-------------------|
| platformer | 横版闯关 | 马里奥、闯关、跳、平台、踩怪 |
| shmup | 街机飞机射击 | 飞机、射击、雷霆、弹幕、打飞机 |
| survivor | 割草肉鸽 | 割草、生存、升级、吸血鬼、打怪变强 |
| pingpong | 乒乓球 | 乒乓球、乒乓、弹球、球拍 |
| fighting | 横版格斗 | 格斗、拳击、对战、打架、双人打 |
| parkour | 跑酷 | 跑酷、一直跑、躲障碍、无尽跑 |
| racing | 欢乐赛车 | 赛车、开车、竞速、跑道、漂移 |

完整词表落盘 `intent_genre_lexicon.json`：

```json
{
  "platformer": { "keywords": ["马里奥", "闯关", "跳平台"], "weight": 1.0 },
  "shmup": { "keywords": ["飞机", "射击", "打飞机"], "weight": 1.0 }
}
```

---

## 5. 算法分期

### P0 · 关键词打分

```text
score[slug] = Σ (keyword in text ? weight : 0)
best = argmax(score)
confidence = best_score / (sum(scores) + ε)
```

- 同分：按 `featured_genres.slugs` 顺序优先 platformer
- `confidence < 0.6`：P0 仍取 best，UI 加一句「我们也可以换别的哦」（P1 改候选卡）

### P2 · LLM 分类

- 输入：用户句 + 7 品类一句话描述
- 输出 JSON：`{ "genre": "...", "confidence": 0.0-1.0 }`
- 校验 genre ∈ 7 slug，否则降级 P0

## 7. 验收标准

| # | 用例 | 期望 |
|---|------|------|
| T1 | 「我想玩马里奥」 | platformer · confidence≥0.7 |
| T2 | 「打飞机」 | shmup |
| T3 | 「割草升级」 | survivor |
| T4 | 「乒乓球」 | pingpong |
| T5 | 「格斗双人」 | fighting |
| T6 | 「跑酷」 | parkour |
| T7 | 「赛车」 | racing |
| T8 | 「随便玩玩」 | 默认 platformer 或追问（P1） |
| T9 | 空字符串 | 400 · 提示重新输入 |
| T10 | 10 句口语集 | ≥8 正确（主规格 §十一） |

测试句集落盘：`config/intent_test_sentences.json`（🔲 待建）。

---

## 8. 非功能需求

- 响应 P95 < 500ms（P0 本地词表）
- 不调用外部 API（P0）
- 日志脱敏：不记录儿童姓名
