# 品类核心参数规格 v1.0

> **设计原则**：每个品类的**核心手感与系统逻辑预制完成**；用户创作时 AI **只改 tuning 层 + 换皮**，禁止重写 core 层。  
> **对齐**：功能点明细 TPL-xx · 自动化执行控制 §4 · Cursor Rules  
> **配置文件**：`templates/{genre}/config/game_config.json`

---

## 一、总体模型：三层参数

```text
templates/{genre}/
├── config/
│   ├── game_config.json      ← 会话实例只改此文件 + assets 路径
│   └── game_config.schema.json
├── core/                     ← 🔒 预制完成，AI 禁止改
│   ├── *.gd                  ← 移动、命中、波次调度等逻辑
│   └── main.tscn
├── data/                     ← 🔒 结构预制，AI 可换表内数值
│   └── defaults.tres
├── assets/                   ← ✅ 按主题替换
└── project.godot
```

| 层级 | 名称 | 谁写 | AI 创作时能改什么 |
|------|------|------|-------------------|
| **core** | 核心逻辑层 | 人工预制 | **禁止改**（移动方式、命中判定、数值公式） |
| **tuning** | 调参层 | 默认值预制 | **主要改这里**（速度、血量、间隔等标量） |
| **theme** | 主题层 | 素材索引预制 | 贴图路径、文案、配色、BGM |

### 1.1 创作时 AI 工作边界

```text
用户说：「我要太空主题的射击游戏，敌人快一点」
    ↓
编排层：genre=shooter → 复制 templates/shooter/ → workspace/{id}/
    ↓
AI 只改：
  - config/game_config.json → tuning.enemy_speed ×1.3
  - theme.asset_pack = "space"
  - theme.display_name / 敌人名称文案
    ↓
禁止改：
  - core/player_controller.gd（移动方式、射击冷却机制）
  - core/hit_detector.gd（命中盒与伤害结算）
```

### 1.2 统一配置文件结构

```json
{
  "meta": {
    "genre": "shooter",
    "template_version": "1.0",
    "session_id": "",
    "display_name": "小行星守卫战"
  },
  "core_locked": true,
  "tuning": { },
  "theme": { },
  "content_safety": {
    "max_violence_level": 0,
    "max_keys": 3
  }
}
```

> `core_locked: true` 时，Cursor Rules 强制 Agent 不得编辑 `core/` 下任何文件。

## 三、用户意图 → 参数映射（编排层）

Kiosk 采集的模糊需求，由编排层转为 **tuning 偏移**，不交给 LLM 自由发挥：

| 用户说法 | 映射动作 |
|----------|----------|
| 「简单一点」 | `difficulty=easy`：敌人 hp×0.7，玩家 hp+1 |
| 「难一点」 | 敌人 speed×1.2，波次+1 |
| 「太空主题」 | `theme.asset_pack=space`，替换 sprite 路径表 |
| 「跳高一点」（闯关） | `jump_velocity × 1.15`，上限 clamp |
| 「塔伤害高」 | `tower.basic.damage × 1.3` |

实现位置：`backend/tuning_mapper.py`（Wave 2 建）

## 四、AI Prompt 约束（写入 prompts/{genre}.md）

```markdown
## 硬性规则
1. 只修改 workspace/{session_id}/config/game_config.json 的 tuning 和 theme 字段
2. 禁止修改 core/ 目录下任何 .gd 文件
3. 禁止修改碰撞层、伤害公式、移动模式枚举
4. 素材路径必须从 assets/kenney/{asset_pack}/ 选取
5. 改完执行 godot-mcp run_project 验证

## 本次用户参数（已由编排层预填）
{tuning_json}

## 你只需要
- 按用户文案微调 tuning 数值（±30% 以内）
- 替换 theme 文案与贴图路径
- 若用户无特殊要求，保持默认 tuning 仅换 theme
```

## 十二、预制小技能（v1.0 · 2026-06-20）

> 目录：[`config/optional_skills.json`](../../config/optional_skills.json)

### 12.1 原则

| 项 | 说明 |
|----|------|
| 逻辑 | **core/skills/** 展前预制，CREATE **不新写**技能 |
| 用户 | 引导流程 **S7** 最多选 **2** 个 |
| AI/配置 | 只写 `tuning.enabled_skills[]` 与 `tuning.skills.{id}.cooldown_scale`（±30%） |
| 键位 | 技能占用操作键时，全游戏仍 **≤3 键**（可与射击/跳跃复用） |

### 12.2 game_config 扩展

```json
{
  "tuning": {
    "enabled_skills": ["dash", "shield_burst"],
    "skills": {
      "dash": { "cooldown_scale": 1.0 },
      "shield_burst": { "cooldown_scale": 0.9 }
    }
  }
}
```

## 十三、玩法子模式（替代跨品类 core 融合 · v1.0）

> 配置：[`config/play_variants.json`](../../config/play_variants.json) · 引导 **S2**

| 机制 | 说明 |
|------|------|
| 单品类 | `meta.genre` 仍为 11 选 1 |
| 子模式 | `meta.play_variant` + `tuning_preset` 切换体验 |
| 跳转 | 用户选「越变越强」而当前在射击 → **改绑** `survivor` 并友好提示 |
| 禁止 | 同时加载两个 templates/ 或改 movement.mode |

**射击示例**：

| 子模式 id | 用户感受 | 系统 |
|-----------|----------|------|
| `wave_adventure` | 打怪闯关 | 留 shooter + 波次 preset |
| `power_growth` | 越变越强 | 跳转 survivor |
| `bullet_hell` | 躲弹幕 | 跳转 shmup |

## 十四、美术与 AI 现场生图（v1.0 · 预留）

### 14.1 双轨素材

| 轨道 | 来源 | CREATE 阶段 |
|------|------|-------------|
| **A · 预置** | `assets/kenney/` | S4/S5 点选 → `theme.*.sprite_path` |
| **B · AI 生图** | 生图 API（待定） | S4/S5 输入描述 → 异步生成 → 失败回退 A |

### 14.2 theme 扩展字段

```json
{
  "theme": {
    "asset_source": "kenney",
    "character": {
      "display_name": "小星",
      "sprite_path": "assets/kenney/space/Ship.png",
      "ai_prompt": "蓝色卡通机器人飞行员，Q版，无武器",
      "ai_image_path": "",
      "ai_gen_status": "skipped"
    },
    "props": [
      { "slot": "weapon", "sprite_path": "...", "ai_prompt": "" }
    ]
  }
}
```

`ai_gen_status`: `skipped` | `pending` | `done` | `fallback_kenney`

### 14.3 约束

- K12 内容安全审核 **先于** 写入 workspace  
- 生图 **不阻塞** S8：可先 Kenney 开玩，生成完成后下局生效（或 D10 后启用）  
- Agent **不得**调用未审核 API；由编排层 `image_gen` 服务统一调用
