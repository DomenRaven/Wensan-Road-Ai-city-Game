# B 链伪工作区 · 多文件树与 B5 剧场 v1.0

> **日期**：2026-06-24  
> **范围**：教育版 Kiosk 左侧「伪 IDE」展示层（不参与 Godot 实际运行）

---

## 一、动机

B5「AI 正在制作」阶段需要更真实的程序员创作感：左侧文件树更大、文件更多，且与七款 L0 模板的真实目录对齐；文件自上而下依次浮现，代码区展示对应真实节选；B6/B7 可点击任意树节点查看全文（展示用）。

---

## 二、架构

```
config/edu_workspace_trees.json     # 每品类 8–9 个路径 + 浮现顺序
        ↓
kiosk/edu/edu-file-tree.js          # 树 UI · 文件夹分组 · 点击/高亮
        ↓
kiosk/edu/code-theater.js           # B5：逐文件 reveal → 预览 API → 打字机
        ↓
GET /edu/preview/{genre}/file       # 只读 templates/{genre}/
GET /sessions/.../workspace/file    # generate 后 config/core（用户 workspace）
```

**原则**：树与预览 API 仅展示；游戏仍由 `generate/v2` 写入 `workspace/` 后 `play/launch` 启动。

---

## 四、B5 时序

1. `analyze-requirements`（右栏进度）
2. 清空并挂载文件树
3. 按 manifest 顺序：文件夹头 → 文件 fade-in → 拉取预览 → 节选行打字（`.gd`≤42 · `.json`≤55 · `.tscn`≤28 行）
4. 全部文件浮现后 → `generate/v2` → B6

总时长预算：`kiosk_edu_spec.json` → `theater.theater_total_ms`（默认 45s）的约 58% 用于多文件剧场。
