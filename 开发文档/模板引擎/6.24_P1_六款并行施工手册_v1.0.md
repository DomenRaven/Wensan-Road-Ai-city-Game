| 完型填空 | `config/creative_templates/{slug}.json` |

## 三、六款 tuning 与 action 对照表

| slug | 填空 tuning（≥3 题） | B7 action_id（≥3） | 技能题（optional_skills） |
|------|------------------------|-------------------|---------------------------|
| **shmup** | `player.speed` · `player.bullet_interval_ms` · `spawn.interval_ms` | `fire` · `hit` · `pickup` | `bomb` / `laser_beam` |
| **survivor** | `player.speed` · `spawn.interval_ms` · `session.duration_sec` | `move` · `level_up` · `pickup_xp` | `magnet` / `nova` |
| **pingpong** | `ball.base_speed` · `rules.points_to_win` · `ai.speed`（非 reaction_ms） | `rally` · `score` · `power_smash` | `power_smash` / `curve_ball` |
| **fighting** | `combat.player_speed` · `combat.light_damage` · `ai.move_speed_scale`（**无 skill_pick**，skills 为空） | `light_punch` · `block` · `special` | — 第四题用 `heavy_damage` |
| **parkour** | `runner.speed` · `jump.velocity` · `obstacle.min_gap_ms` | `jump` · `slide` · `collect_coin` | `double_jump` / `slide` |
| **racing** | `car.max_speed` · `car.turn_speed` · `track.lap_distance_px` | `steer` · `boost` · `lap_complete` | `boost` / `drift_snap` |

默认值以 `templates/{slug}/config/game_config.json` 为准；选项 value 在默认 ±30% 内。

推荐名见 [`6.24_B链教育版用户旅程与需求规格_v1.0.md`](./6.24_B链教育版用户旅程与需求规格_v1.0.md) §B2。

## 四、单款施工工序（Agent 逐步）

1. 读 `templates/{slug}/config/game_config.json` 提取真实 tuning 路径与默认值  
2. 读 `config/optional_skills.json` 中该 slug 技能列表  
3. 对照 `templates/{slug}/core/*.gd` 摘录剧场片段 + 锚点 `line_hint`（只读）  
4. 写 `creative_templates/{slug}.json`（3–4 题 + `name_suggestions`）  
5. 写 `code_anchors/{slug}.json`（≥3 anchor，含 `caption`）  
6. 写 `code_theater/{slug}.jsonl`（`total_ms≈45000`，`anchor_id` 与 anchors 同源）  
7. 写 `templates/_edu/{slug}_hooks.gd`（监听游戏事件 → `EduActionBridge.emit_action`）  
8. `validate_creative_templates.py` → 0 error  
9. 回报施工报告（不改 `edu_workspace.py`，留给集成窗）

**hooks 写法**：复制 `platformer_hooks.gd` 结构；在 workspace 挂到 `main.tscn` 的 `EduHooks`（集成窗注册后由 `generate/v2` 注入）。

## 五、集成窗（窗 7）工序

1. `backend/app/services/edu_workspace.py` → `GENRE_HOOKS` 增加 6 款  
2. `python 05-工具脚本/validate_creative_templates.py` → **7/7 OK**  
3. 新建 `开发文档/模板引擎/评审记录/6.24_creative_templates_校准.md`（每款一行：题数 · tuning · 锚点数 · 剧场行数）  
4. 回写 `6.24_整合任务清单` P1-01~04 · P1-07（hooks 部分）  
5. 可选：`backend/tests/test_creative_{slug}.py` 或参数化单测
