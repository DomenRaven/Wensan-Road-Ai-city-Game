## 一、R2 是什么（30 秒版）

**R2 = 第二轮对齐**：不只改 `game_config.json`，还要把 **玩法、界面、手感** 做到接近秒哒可玩版。

| | v1 转译 | R2 对齐 |
|---|---------|---------|
| 改什么 |  mainly config | core + scenes + assets + config |
| 验收 | 脚本能跑 | 和秒哒左右试玩，看不出明显缺功能 |
| 状态字段 | — | `r2_done` / `r2_pending` / `r1_5_lift`（浅对齐） |

---

## 二、双轨原则（必须遵守）

```
秒哒 H5     →  参考答案（PRD、试玩、constants.ts、美术）
Godot 模板  →  唯一展陈运行时（exe + Kiosk）
game_config →  用户个性化边界（tuning / theme / enabled_skills）
core/       →  从秒哒「抢救」的玩法，写进 GDScript，强类型
```

### 做

- 读秒哒 **PRD** + **`constants.ts`**（或 `constants` 同类文件）
- 在 Godot 回答：怎么赢、怎么输、有哪些实体、数字是多少
- 数值进 `tuning`，贴图/音效进 `theme`，逻辑进 `core/`
- 每完成一款：`godot-mcp run_project` 无 ERROR
- 与秒哒预览 URL 或本地导出 **左右试玩 ≥5 分钟**

### 不做

- **不翻译**秒哒 React / Phaser / TypeScript 源码
- **不**把秒哒 H5 作为展台主程序
- **不**依赖秒哒 API（欠费 · 以落盘为准）
- **不**一次改多个 `templates/{slug}/`（单次会话一品类的 Cursor 规则仍有效）
- **不**做联机、存档、内购、广告

## 三、单款 R2 标准工序（A 档）

### 步骤 0 · 开工前

1. 确认 `config/miaoda_parity_r2_registry.json` 中该 slug 为 `r2_pending`
2. 找到参考源（优先级）：
   - 用户完整 git 导出（如 shmup → `秒哒游戏原型/…`）
   - 否则 → `03-背景与调研/data/秒哒精选源码/{slug}/`
3. 打开秒哒预览或本地 `npx vite` 试玩

### 步骤 1 · PRD P0 勾选表

从 `docs/prd.md` 或 `需求文档.md` 提取 **P0**（无则全篇前 80% 规则），制成勾选表。  
**shmup P0 示例**（其他品类照此格式自建）：

| # | P0 项 | 秒哒有 | Godot 完成后勾选 |
|---|--------|--------|------------------|
| 1 | 开始界面 + 开始按钮 | | |
| 2 | 游戏内分数 HUD | | |
| 3 | 结束界面 + 重开 | | |
| 4 | 玩家拖动 + 自动射击 | | |
| 5 | 普通/快速/重型敌机 | | |
| 6 | 敌机发射子弹 | | |
| 7 | Boss（500 分首次，+1000 循环） | | |
| 8 | 道具 ≥2 种（火力/双发/护盾） | | |
| 9 | 击毁得分 · 类型分值不同 | | |
| 10 | 滚动背景 | | |

**A 档过关**：P0 勾选 ≥ **90%**（10 项中至少 9 项）。

### 步骤 2 · 数值对照

| 秒哒 | Godot |
|------|-------|
| `src/game/constants.ts` | `templates/{slug}/config/game_config.json` → `tuning` |
| `ENEMY_TYPES` / `POWERUP_TYPES` | 必要时 `core/` 读 tuning 字典 |
| 玩家可见标题 | `theme.title` |

- 调整须在 schema 允许范围内（**±30%** clamp，规格文档边界）
- 可先跑：`python 05-工具脚本/miaoda_apply_translation.py --slug {slug}` 作初稿，再人工校对

### 步骤 3 · 美术与音效

| 秒哒 | Godot |
|------|-------|
| `src/game/assets.ts` 引用的图 | `templates/{slug}/assets/` + `theme.*_sprite` |
| Web Audio 合成 | `theme.sounds` + `core/theme_sound.gd`（Kenney ogg） |

- 素材仅用 `assets/` 与 Kenney CC0（及秒哒落盘已核验的 CC0 包）
- 禁止 `.tscn` 内嵌未 junction 的大 PNG

### 步骤 4 · 实现

| 层级 | 内容 |
|------|------|
| `scenes/` | 开始/主游戏/结束 · HUD |
| `core/` | 移动、生成、碰撞、胜负、Boss/道具状态机 |
| `config/` | 用户可调项外置 |

**Godot 工程约束**（与 `.cursor/rules` 一致）：

- GDScript **全部强类型**
- 场景深度 ≤ 5 层 · 单场景节点 ≤ 80
- **禁止改** 其他品类目录 · 本会话只改一个 `templates/{slug}/`

### 步骤 5 · 验收

| # | 检查项 | 命令/方法 |
|---|--------|-----------|
| 1 | MCP 无 ERROR | `godot-mcp run_project` → `templates/{slug}` |
| 2 | config 可调 | 改 `theme.title` / `tuning.player.speed` 可见变化 |
| 3 | 试玩对标 | 秒哒 vs Godot 左右对比，记下剩余 gap |
| 4 | 文档回写 | registry `status` → `r2_done` · gap_summary 更新 |

## 四、B 档浅对齐（**不适用于精选 7 款**）

> **更正（2026-06-23）**：`platformer` · `shmup` · `survivor` · `pingpong` · `fighting` · `parkour` · `racing` **全部为 A 深 R2**。以下 B 档工序仅作历史参考，或用于非精选 L0 品类的应急抬升。

**目标**：不简陋，不求与秒哒完全一致。

1. 加 **开始 / 结束** 场景（可复用 platformer 模式改 theme）
2. 运行 `miaoda_apply_translation.py --slug {slug}`
3. 从 PRD 选 **1 项** 秒哒标志体验实现（如 racing 的道具、fighting 的连招提示）
4. 换 1 组更贴题的 Kenney / 秒哒落盘素材
5. MCP PASS · registry 标 `r1_5_lift`

## 五、秒哒本地 H5 运行（仅作参考试玩）

路径因项目而异。shmup 完整导出：

```powershell

npm install
npx vite --host 127.0.0.1 --port 5173
```

## 六、Cursor 会话规范

1. **一次只做一个 slug**（如本次只做 `shmup`）
2. 生成前：`python 05-工具脚本/query_rag.py "{主题}" -k 5`
3. 用户创作约束仍适用：用户侧最终只改 `game_config.json` 的 tuning/theme/enabled_skills；R2 开发阶段允许改 core/scenes
4. 完成必须 `godot-mcp run_project` 且无 ERROR
5. 收工更新：工作状态记录 · registry · 七天清单勾选
