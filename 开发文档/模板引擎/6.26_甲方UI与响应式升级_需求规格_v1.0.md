## 一、甲方四条需求（原文对齐）

| # | 甲方要求 | 本规格 ID | 优先级 |
|---|----------|-----------|--------|
| R1 | 项目 UI 修改为 **蓝白配色** | **P3-UI-01** | P0 |
| R2 | 前端升级：**横屏、竖屏自适应** | **P3-UI-02** | P0 |
| R3 | 伪编译完成、运行 Godot 时，面板按横竖屏调整位置与大小：**横屏占右半屏 · 竖屏占下半屏** | **P3-UI-03** | P0 |
| R4 | 编写可落地的 **升级方案**，与现有实现对齐；评估 **大模型** 是否必须接入 | **P3-PLAN** | P0（文档） |

**范围**

- **必改**：B 链 `/kiosk/edu/` 全阶段 B0–B7；A 链 `/kiosk/` 主色与触控规格同步（见 FRS-UI-01）。
- **不改**：`templates/{slug}/core/**` · 游戏规则逻辑 · 展陈 P0 已冻结品类 hooks 行为（仅 UI/布局层）。

---

## 二、产品目标

1. 展厅大屏与竖屏 pad 均可完整走通 B0–B7，无横向溢出、无遮挡顶栏。
2. 视觉从「深色 IDE」转为 **明亮蓝白**，符合 K12 教育区品牌与甲方审美。
3. 试玩时 **浏览器工作区** 与 **外置 Godot 窗口** 分区明确：横屏左右分、竖屏上下分，讲解员可同框讲解「代码 vs 游戏」。
4. 方案与代码路径一一对应，便于验收与回退（仍锚定 git 标签 `v1.0`）。

## 三、相对 v1.0 的变更摘要

| 维度 | v1.0（当前） | 6.26 目标 |
|------|-------------|-----------|
| 主色 | 深色 IDE `#1e1e2e` + 星空暗底 | **蓝白** 浅底 + 品牌蓝强调 |
| 布局 | 双栏固定 `45% / 55%` 横排 | **orientation 驱动**：横屏双栏 / 竖屏双行 |
| Godot | `play/launch` 默认位置，右栏仅状态卡 | **launch 携带视口** + Win32 窗口归位（P3-UI-03b） |
| 星空背景 | 深色星云 | 改为 **浅蓝星空** 或弱化叠层（不抢可读性） |
| LLM | B1/B5 关键词 + preset | **本阶段不要求新接 LLM**（见方案文档） |

## 四、用户旅程影响（按阶段）

| 阶段 | R1 蓝白 | R2 自适应 | R3 Godot 分区 |
|------|---------|-----------|---------------|
| B0–B2 单栏 | 表单/芯片/顶栏换色 | 竖屏单列居中；横屏保持 max-width 卡片 | — |
| B3–B4 双栏 | 左工作区白底代码区 | 横屏左右；竖屏上配方下预览 | 右侧/下方为品类动画预览 |
| B5 制作 | 剧场浅底 + 蓝强调 | 竖屏剧场全宽 | — |
| B6–B7 试玩 | launch 卡蓝白样式 | 横屏左代码右引导；竖屏上代码下引导 | **launch 后 Godot 窗口占右半/下半** |

## 五、验收标准（总表）

| # | 项 | 标准 |
|---|-----|------|
| AC-01 | 蓝白主题 | `kiosk_edu_spec.colors` 全站生效；对比度 WCAG AA（正文/按钮） | ✅ P3-1 2026-06-26 |
| AC-02 | 横屏 1920×1080 | 双栏无溢出；触控 ≥48px |
| AC-03 | 竖屏 1080×1920 | 双行布局；顶栏+底栏可点；无横向滚动 |
| AC-04 | 旋转 | `resize` / `orientationchange` 2s 内完成布局切换 |
| AC-05 | Godot 横屏 | 外置窗口左上角 x≈50%屏宽，宽约 50% 屏宽 |
| AC-06 | Godot 竖屏 | 外置窗口 y≈50%屏高，高约 50% 屏高 |
| AC-07 | 回归 | B7 platformer 跳/踩高亮仍可用；`validate` 7/7 不退化 |
| AC-08 | 方案 | [`6.26_甲方升级方案与LLM评估_v1.0.md`](./6.26_甲方升级方案与LLM评估_v1.0.md) 与文件映射一致 |
