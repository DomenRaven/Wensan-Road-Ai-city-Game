## 一、本手册用途

记录 P3-1 蓝白验收后 **讲解员演示区**（B6 未启动 / B7 已启动）修复闭环，供：

- 新增品类时复用 `renderGenreDemoActionsHtml` / `bindGenreDemoActions` 模式  
- 排查「演示钮品类错位」「右栏溢出」「shmup 误留受伤钮」

## 二、架构速查

```text
edu-wizard.js
  renderGenreDemoActionsHtml(genre, { compact, launched })
  bindGenreDemoActions(genre, { launched })  → data-demo-action → simulateAction
  renderPlayReadyPanel() / renderB7Play()     → 外包 .pane-right-stack
        ↓
code-highlight.js → handleAction(action_id) → 切文件 · 黄框 · captionBubble
        ↓
getGenreHighlightFallback(genre) + code_map（B5 generate）
```

| 层级 | 职责 | 本窗常见故障 |
|------|------|--------------|
| **B6 面板** | 未 launch 前讲解员点钮 | 写死 platformer jump/stomp |
| **B7 面板** | launch 后演示钮 | 缺品类分支 → fallback 平台钮 |
| **CSS** | 右栏顶对齐可滚动 | `pane-right-inner` 居中 flex 压扁卡片 |
| **fallback** | 无 code_map 时锚点 | 缺 `level_up` · 文件/行号与 anchors 不一致 |
| **真游玩** | hooks → jsonl → 轮询 | `move` 占线（survivor）· **shmup pickup 被 kill_enemy 展示锁压制**（见 §4.2 S5） |

## 三、根因与标准修复

### 3.1 B6 写死 platformer 钮

| 现象 | 非 platformer 在 B6 显示「模拟跳跃/踩怪」 |
| 修复 | `renderPlayReadyPanel` 改调 `renderGenreDemoActionsHtml(genre)` |

### 3.2 B7 缺品类分支

| 现象 | pingpong / survivor fallback 到 platformer 默认三钮 |
| 修复 | `GENRE_DEMO_ACTIONS` 表驱动七款；B6/B7 共用 |

### 3.3 右栏布局错位

| 现象 | play-ready 卡片偏中/溢出左栏 |
| 修复 | `.pane-right-stack` + `:has(.pane-right-stack)` 顶对齐 `overflow-y:auto` |
| 样式 | `.demo-panel-actions--wrap` 两列换行 · `btn-demo-action` ≥48px |

### 3.4 shmup B7 受伤钮误导

| 现象 | 已 launch 仍显示「受伤」 |
| 修复 | `getGenreDemoActions` 在 `launched && shmup` 时省略 hit；anchors 本就不含 B6 受伤 |

| genre | 按钮 | action_id | 备注 |
