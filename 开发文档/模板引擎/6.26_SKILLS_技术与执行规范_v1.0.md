## 一、施工边界

| 允许 | 禁止 |
|------|------|
| `templates/{slug}/core/skills/*.gd` | 一次会话改多个 `templates/{slug}/` |
| `templates/{slug}/core/*_*.gd` 主玩法接线（用户解冻后） | 改 `templates/{slug}/core/` 移动/命中/公式预制逻辑（非技能门控） |
| `config/code_anchors/{slug}.json` 增技能锚点（解冻后） | 新增 `optional_skills.json` 的 skill_id |
| `kiosk/edu` B7 高亮 / fallback（UI 窗） | 改 `creative_templates` 题目结构（skill_ids 已定时） |
| `templates/_edu/{slug}_hooks.gd` 技能 action 上报（若需要） | 联机/存档/内购 |

---

## 二、技能模块约定

1. **静态类**：`extends RefCounted`，优先 `static func`（参考 platformer/shooter）。
2. **开关**：`is_enabled()` → `GameConfig.has_skill(id)`。
3. **冷却**：`try_activate()` 内读 `GameConfig.get_skill_cooldown_scale(id)`。
4. **输入**：主脚本读 `Input.is_action_just_pressed("skill")`（或品类既有约定）；技能模块不直接绑输入。
5. **未启用**：主脚本在调用前判断 `is_enabled()` 或 `try_activate()` 返回 false，**行为与改前完全一致**。

## 三、UI 对齐（零新增问卷）

| 真相源 | 字段 |
|--------|------|
| `config/optional_skills.json` → `catalog.{genre}` | `id` · `label` · `desc` |
| `config/creative_templates/{genre}.json` | `questions[].skill_ids` 必须与 catalog id 一致 |
| B4 `skill_pick` 组件 | 展示 label，写入 `tuning.enabled_skills` |

**禁止**在 FRS 里发明与 UI 不同的技能名或第三个技能。

## 五、B7 教育（UI 窗）

1. **CORE 窗**可不改 kiosk；用户 Godot 验玩法即可。
2. **UI 窗**须保证：
   - `enabled_skills` 行高亮（generate 后 `code_map`）
   - 可选：技能释放时 `edu_hooks` 上报 `skill_bomb` 等（仅当需黄框联动试玩）
   - 或 B6 演示钮 `simulateAction`（仅演示，非必须）
3. 完整创作流验收：B1→B7 至少 **2~3 遍**，含「不选技能 / 单选 / 双选」。

## 六、Godot 约束

- Godot 4.6 + GDScript **强类型**
- 场景深度 ≤ 5 · 单场景节点 ≤ 80
- 素材仅 `assets/` 与 Kenney CC0
- 收工：`godot-mcp run_project` 无 ERROR（或本地 smoke）
