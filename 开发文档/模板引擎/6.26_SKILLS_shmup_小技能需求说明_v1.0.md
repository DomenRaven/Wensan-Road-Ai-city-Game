## 三、CORE 窗功能需求

### FR-SH-01 清屏炸弹 `bomb`

**触发**：玩家按下 `skill` 动作（与全品类统一技能键，≤3 键约束）。

**前置**：`GameConfig.has_skill("bomb")` 且 `BombSkill.try_activate()` 成功。

**效果**：

1. 调用 `bullet_pool.clear_enemy_bullets()` 清除当前敌弹  
2. 可选：对屏内非 Boss 敌机造成 1 次伤害（与道具强度相当，不秒杀 Boss）  
3. 播放 `ThemeSoundUtil` 爆炸/确认类音效  
4. 进入冷却 15s（× `cooldown_scale`）

**未勾选时**：按 `skill` 键无任何效果（不误触道具逻辑）。

### FR-SH-02 穿透激光 `laser_beam`

**触发**：同上 `skill` 键；与 bomb **互斥单次按键**——若同帧两技能都可释放，优先规则：**先尝试 bomb，否则 laser**（或文档化：先 laser 后 bomb，施工单采用 shooter 顺序：按技能列表顺序尝试）。

**效果**：

1. `LaserBeamSkill.try_activate()` → `_active_remaining = 3s`  
2. 在 active 期间，`spawn_player_bullet(..., pierce=true)`  
3. 激光弹可区分视觉（加粗/modulate，最小改动）  
4. 冷却 6s（× scale），active 内不可再次激活

### FR-SH-03 输入与 tick

参考 shooter `player_shooter.gd`：

- `_physics_process`：`BombSkill.tick(delta)` · `LaserBeamSkill.tick(delta)`  
- `_handle_skills()`：`Input.is_action_just_pressed("skill")`  
- 自动开火逻辑在 laser active 时仍运行，但 `_fire_bullet` 传入 `LaserBeamSkill.is_active()` 决定 pierce

### FR-SH-U02 实现高亮（建议）

在 `code_anchors/shmup.json` 增补（需解冻）：

| anchor_id | file | 说明 |
|-----------|------|------|
| `skill_bomb` | `core/skills/bomb.gd` | 清屏炸弹逻辑 |
| `skill_laser` | `core/skills/laser_beam.gd` | 激光逻辑 |

释放技能时：

- 方案 A：`shmup_hooks.gd` 上报 `action_id: skill_used` + payload（扩展 bridge 非必须）  
- 方案 B：仅依赖 B4 后查看 config + 试玩肉眼验证（最低配）  
- **推荐方案 A 简化**：release 技能时写 `.edu_actions.jsonl` 自定义 action `skill_bomb` / `skill_laser`（若与 B7 占线冲突则仅 UI 演示钮）

## 五、验收标准（CORE 人工）

| # | 条件 | 预期 |
|---|------|------|
| 1 | `enabled_skills: []` | 按 skill 键无效果 |

| 5 | 未改 tuning 其它项 | 移动/射击/道具仍正常 |
