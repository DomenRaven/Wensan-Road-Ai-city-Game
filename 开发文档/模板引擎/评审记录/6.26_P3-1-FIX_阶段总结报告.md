## 一、背景与目标

P3-1 蓝白主题验收后，用户在 B6（未启动试玩）/ B7（已启动试玩）阶段反馈 **讲解员演示区** 存在品类错位、布局溢出、以及割草肉鸽高亮异常等问题（issue 2.1–2.7）。

本阶段目标：

1. 七款精选游戏在 B6/B7 显示**正确品类演示钮**
2. 右栏 play 面板**顶对齐、可滚动、不溢出**
3. 演示钮与真游玩均能触发**左侧代码黄框 + 解说气泡**
4. 不修改 `templates/{slug}/core/**` 与冻结锚点

---

## 二、根因摘要

| 根因 | 影响 |
|------|------|
| B6 `renderPlayReadyPanel` 写死 platformer「跳跃/踩怪」 | 非 platformer 演示钮错位 |
| B7 缺 pingpong / survivor 分支 | fallback 到平台跳跃默认钮 |
| `pane-right-inner` 居中 flex 与 play-ready 冲突 | 右栏视觉错位、横向溢出 |
| shmup B7 保留「受伤」钮 | 与真 Godot 行为不一致，易误导 |
| survivor `move` 占线 + fallback 缺 `level_up` | 吸经验/升级无高亮 |
| `pickup_xp`/`level_up` 优先级高于 `kill_enemy` | 升级后消灭敌人高亮被展示锁永久压制 |

---

## 三、主要交付

### 3.1 前端（`kiosk/edu`）

- 抽取 **`renderGenreDemoActionsHtml`** / **`bindGenreDemoActions`**，B6/B7 共用 `GENRE_DEMO_ACTIONS` 表驱动
- 右栏外包 **`.pane-right-stack`**，CSS 顶对齐 + 演示钮 `flex-wrap` 换行（触控 ≥48px）
- **pingpong** 移除未实装的 `power_smash`（扣杀），仅保留击球/得分
- **shmup** B7 已启动时不渲染「受伤」钮
- **survivor** fallback 补全 `pickup_xp` / `level_up` 锚点，对齐 `code_anchors`

### 3.2 高亮链路

- `code-highlight.js`：`suppress_actions` 加 `move`；`kill_enemy` 优先级升至 **10**；展示锁过期自动清零
- `kiosk_edu_spec.json` 同步上述配置

### 3.3 后端 / Godot 桥（仅 `_edu`，未动 core）

- `survivor_hooks.gd`：停报 `move`；宝石池/升级 UI 延迟重扫
- `edu_action_bridge.gd`：`pickup_xp` / `level_up` 去抖

### 3.4 文档

- [`6.26_P3-1-FIX_讲解员演示区_热修经验手册_v1.0.md`](../6.26_P3-1-FIX_讲解员演示区_热修经验手册_v1.0.md)
