## 一、背景与目标

甲方需求 **R2**：横屏大屏左右双栏讲代码 + 引导试玩；竖屏 Pad 上下双行，上行伪 IDE、下行试玩引导。P3-1 蓝白与 P3-1-FIX 讲解员演示区已冻结，本窗 **仅** 增加方向检测与响应式布局，并为 P3-3 Godot 外置窗贴边预留 `client_viewport` 载荷。

目标：

1. `body[data-orientation="landscape|portrait"]` 单一真相源  
2. 双栏步 B3–B7：landscape 45%/55% 列 · portrait 45%/55% 行  
3. 竖屏 B7 演示区可滚动、演示钮可点、不挡左栏工具栏  
4. 不修改 `GENRE_DEMO_ACTIONS` / `templates/{slug}/core/**`

## 二、主要交付

### 2.1 方向模块（RS-01~02）

新建 `kiosk/edu/layout/orientation.js`：

| API | 行为 |
|-----|------|
| `mount()` | 监听 `resize` / `orientationchange`，防抖 150ms |
| `getMode()` | `"landscape"` \| `"portrait"` |
| `configure(layout)` | 读取 spec `orientation_breakpoint_px`（默认 900） |
| `getViewportPayload()` | 返回 `orientation` + `client_viewport`（screen、kiosk_rect、godot_zone_rect） |

`index.html` 在 `edu-wizard.js` 之前引入；`edu-wizard.js` `init()` 在 bootstrap 后 `configure` + `mount()`。

### 2.2 布局与样式（RS-03~04）

`edu-styles.css`：

- 移除 `@media (max-width:900px)`，改由 `body[data-orientation]` 驱动 grid  
- portrait：`.step-panel` 全宽 padding · `.edu-header` 折行 · `.dual-toolbar` sticky 底栏  
- portrait + play：`.pane-right-inner:has(.pane-right-stack)` 顶对齐可纵滚  
- `.pane-right.godot-zone` 蓝虚线框（与 P3-3 贴边视觉共用）

`dual-pane.js`：`setPhase("play")` 时为右栏加 `godot-zone`。

### 2.3 配置（RS-06）

`config/kiosk_edu_spec.json`：

```json
"layout": {
  "orientation_breakpoint_px": 900,
  "godot_zone": {
    "landscape": { "anchor": "right", "width_pct": 50 },
    "portrait": { "anchor": "bottom", "height_pct": 50 }
  }
}
```

### 2.4 验收期 UX 热修（用户反馈 · 纳入本窗收工）

| 问题 | 处理 |
|------|------|
| 竖屏「完成创作/加载经典版」靠右 | portrait `.dual-toolbar` → `justify-content: center` |
| 竖屏黄色解说气泡贴屏幕最底 | portrait `.caption-bubble` 按 header + 55% 分界定位 |
| 长代码高亮滚动带动整页偏移 | `code-viewer.js` `scrollIntoContainer` 限定 `.code-scroll`；portrait `overscroll-behavior: contain` + 代码区横滚 |
