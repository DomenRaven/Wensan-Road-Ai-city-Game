## 二、技术要点摘要

### E-P0-17 · 真代码

- 删除或降级 `getFallbackConfigSnippet()` 在 **有 workspace** 时的使用
- 新增只读 API（建议）：

```http
GET /sessions/{session_id}/workspace/game-config
→ 200 { "ok": true, "path": "...", "content": "<json string>", "genre": "..." }
```

- `EduCodeViewer.setContent(content, highlightLines)` 用真文本
- 无 workspace → 面板报错「请先完成制作」

### E-P0-18 · 试玩联调

- `launchGame()` 成功：右侧显示「Godot 已启动 · 请到游戏窗口试玩」+ pid/路径（可折叠）
- 失败：红色文案 + 保留经典版按钮
- B7：移除或缩小纯 placeholder；保留模拟按钮为 **讲解员降级演示**
- `EduCodeHighlight.startPolling` 在 launch 成功后启动

### E-P0-19 · 隔离

- 审计 `kiosk/wizard.js` `btnReset` · `pagehide` 是否 release
- 审计 `edu-wizard.js` `btnReset` 与 `EduSession.releaseAsync`
- 脚本：3 session 顺序创建 → A 命名 → release → B 不得读到 A 的 meta
- 展前：`git diff templates/` 必须为空

### D6 · 精选 7 款

- 入口 `/kiosk/` shortcut **S0→S1→S9**（S1 直跳 S9）· **非** edu 路径
- S1 宫格已由 `kiosk_ui_spec.featured_genres` 过滤为 7 款 — 本窗以 **人工试玩表 7/7** 为主
- 模板：[`6.24_展陈P0_D6_七款试玩记录表_v1.0.md`](./6.24_展陈P0_D6_七款试玩记录表_v1.0.md)
