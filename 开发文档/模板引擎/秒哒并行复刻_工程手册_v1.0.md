## 四、单款标准工序（全品类）

### 阶段 0 · 开工（15 min）

1. 确认本窗口 **只改** `templates/{slug}/`
2. 读 `秒哒游戏原型/…/docs/prd.md`
3. 读 `src/game/constants.ts`（或 `constants.js`）
4. 浏览 `src/game/scenes/` 主场景（通常 `Game.ts` / `MainGame.ts`）
5. `npm install && npx vite` 试玩秒哒 5 分钟
6. 填 P0 勾选表（见规范手册 §三 步骤 1）

### 阶段 1 · 资产（30–60 min）

1. 从 `src/assets/` 复制核心 PNG/spritesheet 到 `templates/{slug}/assets/`
2. **文件名含 `[` `]` 必须改名**（如 `ships_packed.png`）→ 用 `Image.load` 或 Godot 导入
3. 记录 manifest：改了哪些路径、帧尺寸

### 阶段 2 · 玩法 core（主要工时）

| 秒哒 | Godot |
|------|-------|
| Phaser Scene | `scenes/game.tscn` + `core/game_area.gd` |
| React UI 叠层 | `scenes/main.tscn` + `core/game_manager.gd` |
| Zustand store | `game_manager` 分数/状态 |
| constants | `config/game_config.json` → `tuning` |
| 碰撞组 | `Area2D` + `collision_layer/mask` |
| 对象池 | `core/*_pool.gd`（子弹/敌/宝石等） |

**时序铁律**（来自 shmup 验收）：

- 动态实体：`add_child` **之后** 再 `configure()`
- 场景 `setup()`：用 `call_deferred`
- 场景用 `preload` PackedScene，避免每次 `load()`
- **禁止** `_game_root.visible` 与 `_begin_start_flow` 不同步（会黑屏）

### 阶段 3 · 验证与回写

1. `godot-mcp run_project` → `templates/{slug}` → **errors: []**
2. 用户试玩：开始 → 核心循环 → 结束/重开
3. 更新 `config/miaoda_parity_r2_registry.json`
4. 有独有坑则写 `{slug}_R2复刻经验手册_v1.0.md`
5. 回写 `工作状态记录_v1.0.md` 一行
