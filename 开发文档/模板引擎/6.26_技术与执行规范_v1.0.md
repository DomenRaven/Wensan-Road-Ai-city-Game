## 二、CSS 与配置规范

1. **色板单一真相**：`config/kiosk_edu_spec.json` → `colors`；CSS `:root` 与 spec 保持同步。
2. **方向单一真相**：`body[data-orientation]` 仅由 `orientation.js` 写入。
3. **禁止使用** `!important` 覆盖（除覆盖第三方一处需注释）。
4. 触控目标 **≥48px** 不变。
5. 蓝白主题下代码高亮块仍需与背景对比度 ≥4.5:1。

---

## 三、响应式断点

| 名称 | 条件 | 布局 |
|------|------|------|
| `landscape` | `innerWidth ≥ 900` 且非 portrait media | 双列 |
| `portrait` | 否则 | 双行 |

`orientation.js` 须在 `edu-wizard.js` init 之前加载。

---

## 四、Godot 窗口归位（Windows 展厅）

1. 仅 **win32** 执行 `SetWindowPos`；其他平台返回 `window_placed: false`。
2. 启动 Godot 后等待 **≥350ms** 再查找 HWND（与现 launcher sleep 对齐）。
3. 查找策略：按 PID 枚举顶层窗口，标题含 `Godot` 或进程可执行名。
4. **不** 强制置顶；避免压住 kiosk 浏览器（kiosk 建议占左半/上半）。
5. launch 请求 **无** `client_viewport` 时保持 v1.0 行为。

## 五、API 契约扩展

### `POST /sessions/{id}/play/launch`

**Request（可选字段）**

```json
{
  "orientation": "landscape",
  "client_viewport": {
    "screen_x": 0, "screen_y": 0, "screen_w": 1920, "screen_h": 1080,
    "kiosk_rect": { "x": 960, "y": 0, "w": 960, "h": 1080 }
  }
}
```

**Response 追加**

```json
{ "window_placed": true, "placement_rect": { "x": 960, "y": 0, "w": 960, "h": 1080 } }
```
