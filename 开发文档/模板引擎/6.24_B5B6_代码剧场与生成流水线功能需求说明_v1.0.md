# 6.24 · B5–B6 代码剧场与生成流水线功能需求说明 v1.0

> **阶段**：B5 需求分析+剧场 · B6 真代码+启动  
> **API**：`POST .../analyze-requirements` · `POST .../generate/v2` · `POST .../play/launch`  
> **资产**：`assets/code_theater/{slug}.jsonl`

---

## 1. B5 · 需求分析

### 1.1 流程

```text
用户点「开始制作」
  → POST analyze-requirements
  → 返回 resolutions（preset | llm_patch）
  → 左栏进入 theater_scrolling
  → POST generate/v2（可与剧场并行）
  → applying → ready
```

### 1.2 analyze 输出

每题一条 resolution：

| 字段 | 说明 |
|------|------|
| `question_id` | 对应 B4 |
| `resolution` | `preset`（P0 仅此）或 `llm_patch`（P2） |
| `tuning_path` | dot path |
| `value` | clamp 后数值或技能 id |
| `code_anchor_id` | 锚点 |

`llm_patch_required: false`（P0 强制）。

### 1.3 降级

- generate 失败：提示 +「加载经典版」按钮（`play/launch` → templates）
- analyze 异常：全用 default 选项

## 2. B5 · 代码剧场

### 2.1 目标

让儿童/讲解员 **看见「AI 在写代码」**；内容 **预制**，非 LLM 实时流。

### 2.2 素材格式 `*.jsonl`

每行：

```json
{"delay_ms": 80, "text": "    \"jump_velocity\": -400,", "highlight": true, "anchor_id": "jump"}
```

| 字段 | 说明 |
|------|------|
| `delay_ms` | 距上一条间隔 |
| `text` | 显示行（可含缩进） |
| `highlight` | 是否强调色 |
| `anchor_id` | 与 `code_anchors` 一致 |

### 2.3 动画参数（`kiosk_edu_spec`）

| 参数 | 默认 |
|------|------|
| `theater_total_ms` | 45000 |
| `line_scroll_speed` | 匀速 · 无跳跃 |
| `background` | `#282a36` |
| `keyword_color` | `#ff79c6` |
| `string_color` | `#f1fa8c` |
| `sound_typing` | 可选 · 默认关 |

### 2.4 状态机

| 状态 | 左栏显示 |
|------|----------|
| `theater_scrolling` | jsonl 逐行追加滚动 |
| `applying` | 进度条「正在写入你的游戏…」 |
| `ready` | 真实 `game_config.json` 片段 + 1 个 core 摘要 |

### 2.5 同源约束

剧场中出现的 `jump_velocity` 行 **必须** 与 B7 高亮、`code_anchors.jump` 一致。

## 3. generate/v2 流水线

### 3.1 步骤

1. 校验 `creative_answers` + analyze 完成
2. `atomic_copytree` `templates/{genre}/` → `workspace/{session_id}/`
3. 按 resolutions merge `config/game_config.json`（`config_builder` + clamp）
4. 复制 `templates/_edu/edu_action_bridge.gd` 等（若启用 B7）
5. 扫描文件生成 `code_map`（真实行号）
6. `session.phase = PLAY` · 返回路径

### 3.2 与 v1 差异

| 项 | v1 | v2 |
|----|----|----|
| 前置 | recap_confirmed | creative_answers |
| 输出 | workspace | + code_map |

旧向导继续调用 v1；教育版 **仅 v2**。

## 4. B6 · 真代码展示

### 4.1 左栏

- 默认展开 `config/game_config.json`
- 高亮用户改过的 tuning 行（对比模板默认值 diff）
- 可选第二 tab：`core/player_*.gd` 摘要 30 行

### 4.2 右栏

- 文案：「这就是 AI 根据你的选择生成的游戏！」
- 按钮「开始试玩」→ `POST /play/launch`
- Godot 窗口 contain 适配

### 4.3 验收

- [ ] `godot-mcp run_project` workspace 无 ERROR
- [ ] 剧场 45s 内可播完或可配置循环
- [ ] 剧场期间 UI 不黑屏、不卡死
- [ ] 真代码与 workspace 文件一致
