## 一、根因摘要

| 根因 | 影响 |
|------|------|
| `kill_enemy` 优先级 **10** + 展示锁约 5s，`pickup` 原 **9** | 实机持续击毁敌机时，吃道具高亮被展示锁**永久压制** |
| `shmup_hooks` 对新生道具 `call_deferred("_wire_powerups")` | 敌机刚爆、道具刚掉即被吃到时，钩子**未连上** `collected`，`.edu_actions.jsonl` 无 `pickup` |
| `getMergedCodeMap` 后者覆盖：`pickup` tuning 行盖掉 `apply_powerup` | 偶发锚到 `game_config.json` 而非 `core/player_ship.gd` |
| 旧 workspace 内 hooks 副本过期 | 仅改 `_edu` 源文件时，未重新 B5 的 session 仍跑旧 hooks |

**非根因**：演示钮逻辑（P3-1-FIX 已修）· 锚点 JSON（未改 `code_anchors/shmup.json`）
