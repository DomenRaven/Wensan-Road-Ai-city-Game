## 一、硬约束

| # | 规则 |
|---|------|
| 1 | **禁止**修改 `templates/{genre}/core/**` |
| 2 | **禁止**修改 `analyzer.py` / `generate` 主流程（clamp 行为保持） |
| 3 | **禁止**改 `optional_skills.json` 的 skill ID |
| 4 | 单次会话优先 **一个施工窗** 完成 RECIPE-B 全七款 **或** 按冻结品类分批（需用户解冻声明） |
| 5 | `tuning` 新值须在规格文档表内 ±30%，不得越界 |
| 6 | 面向 14 岁以下：证书无外链二维码追踪（可选静态文案即可） |

## 二、冻结品类解冻（RECIPE-B）

七款 `creative_templates` 多数在冻结规则内。施工前：

| 品类 | 解冻文件（典型） |
|------|------------------|
| shmup · pingpong | `creative_templates` + `code_anchors`（共享 manifest） |
| platformer · parkour · racing · fighting | 各 `frozen_genre_*.json` 对应 config 路径 |
| survivor | 通常未单品类冻结 · 仍谨慎 diff |

解冻后 **只改 config + kiosk**；**不** `freeze_* write` 模板 core（因 core 未动）。

## 三、问卷 JSON 规范

```json
{
  "id": "q_hp",
  "widget": "single_choice",
  "prompt": "你有几条命？",
  "options": [
    { "id": "low", "label": "少一点", "tuning_path": "tuning.player.max_hp", "value": 2, "code_anchor_id": "player_hp" },
    { "id": "default", "label": "刚刚好", "tuning_path": "tuning.player.max_hp", "value": 3, "code_anchor_id": "player_hp" },
    { "id": "high", "label": "多一点", "tuning_path": "tuning.player.max_hp", "value": 5, "code_anchor_id": "player_hp" }
  ]
}
```

- 删除 `q_skill` 时去掉整对象，注意 JSON 尾逗号  
- `fighting` 无删项；增题可选

## 四、证书前端规范

- 纯客户端组装；不存 PII 到服务端  
- 打印用 CSS `@media print`，不依赖后端 PDF  
- 样式变量优先 `kiosk_edu_spec.json` → `session.applyThemeFromSpec` 已有色板  
- `certificate.js` 导出挂 `window.EduCertificate`
