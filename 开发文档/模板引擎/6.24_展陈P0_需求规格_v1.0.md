## 一、范围一句话

## 三、分项需求

### D6 · Kiosk 精选 7 款宫格联调

| 项 | 要求 |
|----|------|
| 入口 | `http://127.0.0.1:8080/kiosk/` · shortcut S0→S1→S9 |
| 范围 | **仅** `miaoda_parity_r2_registry` 中 `r2_done` 的 7 款 |
| 验收 | 每款人工 ≥3min · MCP/launch 无阻塞 ERROR · 记录表 7 行 |
| 非目标 | 11 L0 全扫 · tower_defense 等 C 流水线品类 |

### E-P0-17 · B6/B7 左侧真代码

| 项 | 要求 |
|----|------|
| 现状 | `edu-wizard.js` 使用 `getFallbackConfigSnippet()` 硬编码片段 |
| 目标 | generate 成功后，左侧 `code-viewer` 展示 **真实** `game_config.json` 文本 |
| 数据源 | `workspace/{session_id}/config/game_config.json`（只读） |
| 实现 | 推荐 `GET /sessions/{id}/workspace/game-config`（或等价只读 API）· 前端替换 fallback |
| 高亮 | `code_map` 行号与真文件行号一致（已有 generate/v2 返回） |
| 失败 | 无 workspace 时明确报错，禁止静默 fallback 冒充真文件 |

### E-P0-18 · B7 试玩展陈联调

| 项 | 要求 |
|----|------|
| 现状 | B7 右侧「游戏试玩区」为 placeholder + 模拟按钮 |
| 目标 | 点击「开始试玩」→ `POST /play/launch` → 网页显示 **已启动/进程状态/聚焦提示** |
| Godot | **外置窗口**（非 iframe 内嵌）为展陈方案；网页负责状态与左侧高亮 |
| 联动 | launch 成功后 `code-highlight` 开始轮询；失败显示可理解文案 |
| 降级 | launch 失败可保留「加载经典版」但须 log + 讲解员可见提示 |

### E17 · 人工展厅试玩（platformer）

| 项 | 要求 |
|----|------|
| 路径 | `/kiosk/edu/` → platformer 全流程 → B7 真机 Godot |
| 操作 | 跳跃、踩怪各 ≥1 次触发左侧高亮 |
| 时长 | 连续试玩 ≥120s |
| 证据 | [`评审记录/E17_展厅试玩_platformer.md`](./评审记录/E17_展厅试玩_platformer.md) 签字表 |
| 自动化 | 不替代人工；可附 `GET /play/actions` 日志截图 |

### E-P0-19 · 鲁棒性 · 启停复位 · 隔离

| 项 | 要求 |
|----|------|
| bootstrap | `GET /bootstrap` 清理孤儿 workspace · 7 款模板校验 |
| release | 页关闭 sendBeacon · `POST /release` · A 链 `wizard.js` 与 edu `session.js` 均有效 |
| 重新开始 | edu「重新开始」· A 链「重新开始」清空 session/界面状态 |
| 隔离 | 用户写入 **仅** `workspace/{session_id}/` · `templates/{slug}/` **零 diff** |
| 串台 | 连续 3 个虚拟用户：用户 B 不得看到用户 A 的 `display_name` / tuning |
| 脚本 | 推荐 `05-工具脚本/e2e_exhibition_isolation.py` 自动化 3 用户链 |
