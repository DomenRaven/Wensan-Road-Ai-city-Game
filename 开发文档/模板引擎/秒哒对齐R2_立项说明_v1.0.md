## 一、用户意见（原文意图）

> 需要**大幅度向秒哒的版本进行对齐**。无论是**交互逻辑、数字设计，还是美术素材**等。  
> 本地此前生成的 Godot L0 原型**质量不足**，不能作为展陈交付标准。  
> 仅改 `game_config.json` 的 v1 自动转译**看不出差别**，**不作为完成标准**。

**补充约束（项目既有）**

- 秒哒账户**欠费**：不再依赖 API 续生成；以 `03-背景与调研/data/秒哒精选源码/` 落盘 + PRD 为准。
- D10 展陈（6/30）仍以 **Godot + Kiosk** 交付，秒哒 H5 为**设计与手感标杆**，非替代运行时。

## 二、v1 转译 vs R2 对齐

| 维度 | v1 转译（已完成） | R2 对齐（本立项） |
|------|------------------|-------------------|
| 改动范围 | 仅 `game_config.json` | `core/` + `scenes/` + `assets/` + config |
| 目标 | 数值初稿 | **玩法/UX/美术接近秒哒可玩版** |
| 验收 | 脚本 smoke | **左右试玩秒哒 H5 vs Godot** |

v1 产出保留：`*.pre_miaoda_translation` 备份、`miaoda_translation_worksheet_*.json`、`translation_summary.json`。

## 三、platformer 差距快照（首期）

对照 `秒哒精选源码/platformer/docs/需求文档.md` 与 `templates/platformer/`：

| 秒哒有 | 本地 L0 现状 |
|--------|-------------|
| 开始 / 结束 / 胜利界面 | 无，进场景即玩 |
| 3 命、分数、关卡号、金币数 HUD | 仅「星星 x/5」 |
| 踩头杀怪；侧面碰怪扣命 | 碰怪即死，无踩头 |
| 金币 10 分、踩怪 100、通关 500 | 收集星星达标即胜 |
| 砖块、管道、终点旗、卷轴相机 | 固定短关卡、无卷轴 |
| Pixel Adventure 跑动/跳跃动画 | Kenney 单帧 Sprite |
| WASD + 方向键 + 空格 | 部分仅有方向键 |

秒哒素材路径见落盘 `src/**/game/assets.ts`（**Pixel Adventure 1 Free**）；本地 `assets/` 尚无该包，R2 需下载合法素材或选最接近 Kenney 包。

## 四、R2 执行规则

1. **单次会话只做一个** `templates/{slug}/`
2. **允许改** `core/`、`scenes/`、`assets/`、`config/game_config.json`
3. **禁止** 联机 / 存档 / 内购；面向 K12；操作 ≤ 3 键
4. 强类型 GDScript；场景深度 ≤ 5 层；单场景节点 ≤ 80
5. 完成必须 `godot-mcp run_project` 无 ERROR
6. 每阶段与秒哒预览 URL 对比试玩（欠费下预览若仍可用）

---

## 五、品类顺序

`platformer` → `survivor` → `racing` → `fighting` → `pingpong` → `shmup` → `parkour`

展陈若时间紧：**platformer + survivor + pingpong** 优先做到 R2，其余维持 L0 快速试玩。
