## 一、甲方 / 产品目标

| # | 需求 | 验收 |
|---|------|------|
| R-S1 | 孩子在 B4 勾选的小技能，B7 试玩可**感知** | 开/关对比明显 |
| R-S2 | 技能 ID、文案与 UI **一致** | 以 `optional_skills.json` + `creative_templates` 为唯一真相 |
| R-S3 | 不改创作问卷结构（减少工作量） | 不新增 `skill_ids`，先接线已有项 |
| R-S4 | B7 可展示 `enabled_skills` 与技能实现锚点 | 黄框 + 代码滚动（UI 接线窗） |
| R-S5 | 每品类独立收工 + 经验文档 | 供后续六款复制 |

---

## 二、范围

### 在范围

- 展厅 **7 款**：platformer · shmup · survivor · pingpong · fighting · parkour · racing  
- `templates/{genre}/core/skills/` + 主玩法脚本接线  
- `config/code_anchors` 增补（仅技能相关锚点，需解冻时用户确认）  
- `kiosk/edu` B7 高亮 / 演示（UI 接线子窗）  
- 文档库、冻结快照、`freeze_* verify`

### 不在范围

- 新增技能 ID（展后单独立项）  
- 改 `analyzer` / `generate` 主流程  
- LLM 生成技能逻辑  
- 改 P3-3 Godot 窗口分区（并行不混同窗改 `play.py` 归位逻辑）

## 三、分期交付（每品类）

```text
窗 SKILLS-{genre}-CORE   Godot 接线 + Godot 人工验
        ↓
窗 SKILLS-{genre}-UI     B7 锚点/高亮/演示（可选 simulateAction）
        ↓
窗 SKILLS-{genre}-E2E    完整创作流程 2~3 遍（用户）
        ↓
经验手册条目 + 整合清单勾选
```

**当前首推**：`shmup`（战机）→ 见 [`6.26_SKILLS_shmup_小技能需求说明_v1.0.md`](./6.26_SKILLS_shmup_小技能需求说明_v1.0.md)

## 四、技能真相源（全品类）

| 层级 | 文件 | 职责 |
|------|------|------|
| 目录 | `config/optional_skills.json` | 合法 ID · 展厅文案 · 每品类最多 2 个/局 |
| 问卷 | `config/creative_templates/{genre}.json` | `skill_pick.skill_ids` 必须与目录一致 |
| 默认 | `templates/{genre}/config/game_config.json` | `enabled_skills: []` · `skills.{id}.cooldown_scale` |
| 模块 | `templates/{genre}/core/skills/{id}.gd` | `is_enabled` / `try_activate` / `tick` |
| 接线 | `core/player_*.gd` 等主脚本 | 输入 + 效果 |
| 教育 | `config/code_anchors/{genre}.json` | `enabled_skills` + 可选 `skill_*` 锚点 |
