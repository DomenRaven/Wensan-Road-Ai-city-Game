## 二、v1.0 独立重评得分

### 2.1 分维度评分

| 维度 | 权重 | v1.0 得分 | 等级 | 主要依据（可核对） |
|------|------|-----------|------|-------------------|
| **Godot+Cursor+MCP 技术可行性** | 22% | **78** | 可执行 | MCP `get_godot_version` 已通过；社区有 godot-mcp、godot-cursorrules、俯视角/塔防开源模板；**.tscn 手改仍高风险**（调研未否定，无反证） |
| **core/tuning/theme 架构** | 18% | **86** | 优秀 | 与 data-driven design、Game Jam 模板、MDA「Mechanics 锁定」文献一致；调研整合 §一 明确支持 |
| **10 分钟 SLA 可达成性** | 18% | **58** | 需修正 | **无一次 E2E 实测**；7 品类×4 周对单人过乐观；文献支持的是「预制模板」而非「AI 现场写系统」 |
| **展陈/K12 适配** | 12% | **70** | 有风险 | Serious Games 方向正确；但内容审核、双工位队列、讲解员 SOP **仅有文档无实现** |
| **工程复杂度控制** | 12% | **55** | 需修正 | v1.0 同时规划 FastAPI+Redis+Socket+React+MinIO+Cursor SDK；**对「本地开发、操作简单」目标过度设计** |
| **文档完备性** | 8% | **82** | 良好 | 开发文档体系完整；**templates/ 尚无运行代码** |
| **成本与硬件** | 5% | **74** | 可接受 | ¥27k 合理；并发 Cursor+Godot 导出时 **4060 可能吃紧**（待压测，非幻觉） |
| **可维护与降级** | 5% | **68** | 有风险 | 降级链已写清；`demo_preset`、自动修复循环 **未实现** |

### 2.2 加权总分

```
78×0.22 + 86×0.18 + 58×0.18 + 70×0.12 + 55×0.12 + 82×0.08 + 74×0.05 + 68×0.05
= 17.16 + 15.48 + 10.44 + 8.40 + 6.60 + 6.56 + 3.70 + 3.40

```

**结论（独立判断）**：v1.0 方案**方向正确、架构优秀**，但作为可执行工程计划 **尚未达到 75 分「放心开工」线**。主要扣分在 **SLA 未验证** 与 **栈过重**，而非 Godot/Cursor 选错。

## 三、调研对关键论断的证伪/证实

| 方案论断 | 评审结论 | 证据 |
|----------|----------|------|
| Godot 最适合 AI+MCP 本地开发 | **证实** | 全文本项目；GitHub 大量 template；B 站 godot-mcp 教程；已本地跑通 4.6.3 |
| 10 分钟可完成个性化小游戏 | **有条件成立** | 仅当 core 预制 + 只改 config；**不成立**若 Agent 改 core 或重写系统 |
| FastAPI+React Kiosk 是 MVP 必需 | **不成立** | 调研未显示展陈 MVP 依赖该栈；增加单人 2 周+ 工作量 |
| 7 品类 4 周可全部达标 | **存疑** | rogal01 塔防 case、brettchalupa platformer 等单个模板需数天打磨手感 |
| HTML 仅降级 | **合理** | 与用户偏好一致；Godot Web 导出非主路径 |
| GamingAgent 可直接用于展陈 | **不成立** | ICLR 研究框架，非产品化 Godot 流水线 |
| 爬虫结果可直接喂给 AI | **不成立** | 1024 条中混有噪声（无关 GitHub 仓库）；**必须经 RAG 过滤** |

## 四、必须修正项（v1.1 已采纳）

| 优先级 | 问题 | 修正策略 |
|--------|------|----------|
| P0 | 工程栈过重 | **Phase 1** 仅 Godot+Cursor+MCP+RAG；Kiosk/编排推迟 |

| P1 | 7 品类并行 | **MVP 3 品类**：platformer / shooter / tower_defense |
| P1 | Agent 幻觉 | 强制 **query_rag.py** + 只改 `game_config.json` |
| P2 | 展陈 UI |  interim：**Godot 菜单场景** 选品类，优于先做 React |
| P2 | 并发性能 | 展陈 PC 建议 **RTX 4060 16G+**，双 Session 需实测 |
