## 三、观众互动方式详解

### 3.1 需求采集（INTENT）

| 步骤 | 观众感知 | 背后动作 |
| --- | --- | --- |
| ① | 立式屏欢迎「小小游戏设计师」 | Kiosk IDLE 轮播 |
| ② | 点选「太空射击」等品类 | POST `/session` genre=shooter |
| ③ | 选主题「小行星」 | 更新 session theme |
| ④ | 选「小学·简单」 | grade + difficulty |
| ⑤ | 点「开始创作」 | → CREATE；触发 Cursor Prompt |

### 3.2 AI 创作（CREATE）— Phase 1 本地流程

| 步骤 | 操作者感知 | 背后动作（P1） |
| --- | --- | --- |
| ⑥ | Cursor 显示「匹配模板」 | 复制 `templates/{genre}` → `workspace/{id}` |
| ⑦ | 「检索知识库…」 | `query_rag.py` 填充依据 |
| ⑧ | 「更新配置…」 | **仅改** `config/game_config.json` tuning/theme |
| ⑨ | 「试运行…」 | godot-mcp `run_project` |
| ⑩ | 若有报错自动修复 | `get_debug_output` → fix ≤2 轮 |
| ⑪ | 「完成」 | run 无 ERROR 或回退 demo_preset → PLAY |

> Phase 3 展陈时，⑥–⑪ 映射至横屏可视化 + FastAPI 触发，逻辑不变。

### 3.3 试玩与交付（PLAY → DONE）

| 步骤 | 观众感知 | 背后动作 |
| --- | --- | --- |
| ⑪ | 全屏进入游戏 | 启动 exe/Godot |
| ⑫ | 玩 2–3 分钟后点结束 | 屏录高光可选 |
| ⑬ | Kiosk 显示二维码 | MinIO 签名 URL |
| ⑭ | 可选留昵称上墙 | 审核队列 → 背墙轮播 |
