## 一、项目是做什么的

| 项 | 内容 |
|----|------|
| 名称 | AI 小游戏创作工坊 / GameForge K12 |
| 甲方 | 西湖文旅 |
| 场地 | 文三·中国 AI 新源地 — **主厅 AI 教育区**（K12 版块） |
| 参观者 | 6–15 岁学生、亲子家庭、研学团 |
| 时长 | 单人约 **8–10 分钟** |
| 核心能力 | 对话描述 → AI 生成 Godot 小游戏 → 当场试玩 → 扫码带走 |
| 游戏品类 | 射击、塔防、体育竞速、横版闯关、生活模拟、推理解谜、休闲 |
| 预算参考 | 约 **¥27,100**（见技术选型第八章） |

**不要做成**：纯 HTML 网页小游戏、3A 游戏开发课、无护栏开放聊天。

**要做成**：**AI 创作透明厨房** — 看得见 AI 如何选模板、写代码、跑游戏。

## 六、怎么用 Cursor / godot-mcp

### 6.1 环境确认

```powershell
node --version    # ≥18
# Godot 4.6.3 已安装于 F:\Godot\...
```

### 6.2 MCP 已配置

路径：`.cursor/mcp.json`  
验证：在 Agent 对话执行 `get_godot_version` → 应返回 `4.6.3.stable...`

### 6.3 典型 Agent 指令

```
请基于 templates/platformer，在 workspace/demo-001/ 创建太空主题闯关游戏，
替换 Kenney 素材，用 godot-mcp 运行并修复报错，导出到 exports/demo-001/
```

### 6.4 讲解员复位

- 长按 Kiosk「重新开始」→ 清空 Session  
- 异常时播放 `templates/*/demo_preset` 预录视频

## 十、常见问题

**Q：为什么不用 HTML 小游戏？**  
A：甲方偏好原生体验；HTML 仅作无法安装时的降级，性能和展陈质感较弱。

**Q：10 分钟够吗？**  
A：靠预制模板 + 参数化；不做新系统。超时回退 Demo。

**Q：和 SamThree 什么关系？**  
A：同 AI 教育动线；SamThree 学「拆问题」，本模块学「创造」。

**Q：Cursor 要观众看见吗？**  
A：不要。观众看「过程可视化屏」；Cursor 在后台主控 PC 运行。
